title: Compare
description: Where karcarthy fits among agent SDKs, CLIs, and orchestration frameworks.
---

karcarthy is a thin orchestration layer. It is not a replacement for agent
SDKs; it is a way to keep multi-agent coordination inspectable while delegating
single-agent execution to systems that already run model calls and tools.

## Positioning

| Need | Use karcarthy when | Prefer another system when |
| --- | --- | --- |
| Agent loop | You already have an SDK, CLI, or command process that can run one agent. | You need a batteries-included agent runtime first. |
| Workflow shape | You want the graph to be EDN/JSON data that can be stored, generated, diffed, and rewritten. | You want workflows only as Python/TypeScript objects. |
| Tools | Tool execution belongs to Claude CLI, OpenAI Agents SDK, MCP config, or your command adapter. | You want karcarthy itself to define and host tool schemas. |
| Multi-agent patterns | You want `pipe`, `map`, `reduce`, `iterate`, `bind`, or state graphs as portable values. | You want a mature framework-native graph runtime and UI. |
| Testing | You want deterministic offline orchestration tests with `mock-adapter`. | You need provider-level simulation of model/tool behavior. |

## Comparison

| System | Strong at | karcarthy relationship |
| --- | --- | --- |
| Claude CLI / Claude Agent SDK | Running single agents with tools, MCP, sessions, and CLI ergonomics. | Use as an adapter. |
| OpenAI Agents SDK | Agent loop, handoffs, tools, tracing, and structured SDK runtime. | Use as an adapter or mirror routing with `bind`. |
| LangGraph | Durable state graphs with nodes, edges, and transitions. | Port finite graph shapes with `state-graph` when you need data-first representation. |
| CrewAI | Role/task/crew abstractions for collaborative agent work. | Port task crews with `crew`, which compiles to `pipe` or `map`. |
| AutoGen | Multi-agent chat and speaker selection. | Port deterministic round-robin chats with `group-chat`; use `bind` for dynamic speaker choice. |
| DSPy | Signatures, metrics, and prompt/program optimization. | Complementary; karcarthy does not optimize prompts today. |

## Design choice

The important split is:

```text
karcarthy: workflow data, routing, fan-out, loops, result shape
adapter:   model calls, tool calls, sessions, provider-specific runtime
```

That split lets the same workflow run offline, through Claude CLI, through the
OpenAI Agents SDK bridge, or through a command process.
