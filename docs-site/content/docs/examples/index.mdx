title: Run Examples

---

The examples live in the repository `examples/` directory. The Clojure and JVM
examples call the library directly. Python and TypeScript use the JSON bridge
because a workflow is just data.

For a walkthrough of the same non-trivial workflow in Clojure, Python, and
TypeScript, start with [Launch Brief](/docs/guides).

## Clojure

```bash
clojure -M -m karcarthy.demo
clojure -M -e '(load-file "examples/clojure/mappings.clj")'
clojure -M -e '(load-file "examples/clojure/launch.clj")'
```

Runnable framework-mapping examples:

- `examples/clojure/swarm/`
- `examples/clojure/crewai/`
- `examples/clojure/autogen/`
- `examples/clojure/langgraph/`

Live examples are marked explicitly:

- `examples/clojure/live.clj`
- `examples/clojure/evolve.clj`

Guide example:

- `examples/clojure/launch.clj`

## Python

```bash
python3 examples/python/demo.py
python3 examples/python/launch.py
python3 examples/python/swarm/demo.py
python3 examples/python/crewai/demo.py
python3 examples/python/autogen/demo.py
```

The Python examples build workflow dictionaries, send them to `karcarthy.cli`,
and read result JSON.

Guide example:

- `examples/python/launch.py`

## TypeScript

```bash
npx tsx examples/typescript/demo.ts
npx tsx examples/typescript/launch.ts
npx tsx examples/typescript/swarm/demo.ts
npx tsx examples/typescript/crewai/demo.ts
npx tsx examples/typescript/autogen/demo.ts
```

The TypeScript examples build the same data shape as the Python examples.

Guide example:

- `examples/typescript/launch.ts`

## Java, Kotlin, and Scala

Use the Clojure Java API from each JVM language:

```bash
CP=$(clojure -Spath)
javac -d /tmp/karc -cp "$CP" examples/java/Demo.java
java -cp "$CP:/tmp/karc" Demo
```

Kotlin and Scala use the same classpath pattern:

```bash
kotlinc -cp "$CP" examples/kotlin/Demo.kt -include-runtime -d /tmp/demo.jar
java -cp "$CP:/tmp/demo.jar" DemoKt

scalac -cp "$CP" -d /tmp/karc-scala examples/scala/Demo.scala
java -cp "$CP:/tmp/karc-scala" Demo
```
