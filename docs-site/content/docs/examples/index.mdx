Use examples when you want to see a complete command, expected output, and the
workflow idea it demonstrates. Start with the quick checks, then move to the
launch-readiness tutorial when you want a realistic multi-agent workflow.

## Quick checks

| Goal | Command | What it proves |
| --- | --- | --- |
| Run one CLI agent | `./bin/karcarthy agent echo --instructions "Echo the input." hi` | The standalone CLI works and returns text by default. |
| Run a workflow file | `./bin/karcarthy run examples/workflows/echo.json hi` | Workflow data can live outside Clojure. |
| Run the bundled Clojure demo | `clojure -M -m karcarthy.demo` | The library runs offline through the mock adapter. |
| Rewrite workflow EDN | `clojure -M -e '(load-file "examples/clojure/rewrite.clj")'` | Clojure code can transform workflow data before execution. |

Expected output for the first two commands:

```text
[echo] hi
```

## Full workflow tutorial

The launch-readiness examples classify a request, run product/engineering/
security/support reviewers, synthesize their notes, and iterate with a critic.

| Language | Command | Use this when |
| --- | --- | --- |
| Clojure | `clojure -M -e '(load-file "examples/clojure/launch.clj")'` | You want host reducers and direct API usage. |
| Python | `python3 examples/python/launch.py` | You want to generate workflow JSON from application code. |
| TypeScript | `npx tsx examples/typescript/launch.ts` | You want the same JSON workflow shape from a TS app. |

All three default to deterministic mock responses, so they run without model
credentials. See [Tutorials](/docs/tutorials) for the walkthrough.

## JVM language interop

Java, Kotlin, and Scala can call the Clojure API through the Clojure Java API.
The Java example is the verified baseline:

```bash
CP=$(clojure -Spath)
javac -d /tmp/karc -cp "$CP" examples/java/Demo.java
java -cp "$CP:/tmp/karc" Demo
```

Expected output:

```text
ok?  true
text [summarizer] [researcher] what is a monad?
```

Kotlin and Scala use the same classpath pattern:

```bash
kotlinc -cp "$CP" examples/kotlin/Demo.kt -include-runtime -d /tmp/demo.jar
java -cp "$CP:/tmp/demo.jar" DemoKt

scalac -cp "$CP" -d /tmp/karc-scala examples/scala/Demo.scala
java -cp "$CP:/tmp/karc-scala" Demo
```
