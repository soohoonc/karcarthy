---
title: Workflow Data
description: EDN shapes for agents, workflows, and results.
---

Workflows are EDN values. Every node is a map tagged with `:karcarthy/type`.

## Agent

```clojure
{:karcarthy/type :agent
 :name "researcher"
 :instructions "Research the question and cite sources."
 :model "sonnet"
 :tools ["mcp__feedback__search"]
 :adapter :claude}
```

Required keys: `:name`, `:instructions`.

## pipe

Runs each step in order. The `:text` from one result becomes the input for the
next step.

```clojure
{:karcarthy/type :pipe
 :steps [WORKFLOW ...]}
```

## map

Runs a collection of branches on the same input:

```clojure
{:karcarthy/type :map
 :branches [WORKFLOW ...]
 :max-concurrency 16}
```

Or runs a worker once per subtask from a planner:

```clojure
{:karcarthy/type :map
 :planner PLANNER
 :worker WORKFLOW
 :max-concurrency 16}
```

`PLANNER` is a workflow whose result text is EDN:

```clojure
{:subtasks ["first" "second"]}
```

## reduce

Runs mapped work, then passes an EDN summary to a reducer workflow.

```clojure
{:karcarthy/type :reduce
 :mapped {:karcarthy/type :map ...}
 :reducer WORKFLOW}
```

The reducer receives EDN shaped like:

```clojure
{:input "original input"
 :subtasks ["first" "second"]
 :results [{:agent "worker" :ok? true :text "..."}]}
```

## iterate

Runs a worker and evaluator until accepted or the round limit is reached.

```clojure
{:karcarthy/type :iterate
 :worker WORKFLOW
 :evaluator EVALUATOR
 :max-rounds 3}
```

`EVALUATOR` is a workflow whose result text is an EDN verdict:

```clojure
{:accept? true}
{:accept? false :feedback "specific revision feedback"}
```

## bind

Routes by the source result:

```clojure
{:karcarthy/type :bind
 :source WORKFLOW
 :routes {:label WORKFLOW}
 :default WORKFLOW}
```

Or continues from one workflow to another:

```clojure
{:karcarthy/type :bind
 :source WORKFLOW
 :to WORKFLOW
 :prompt "optional override"}
```

When `:routes` is present, `:source` replies with EDN `{:route :label}` and the
selected workflow receives the original input. Route matching is exact. When
`:to` is present, the second workflow receives the source result text unless
`:prompt` is provided.

## Result

```clojure
{:karcarthy/type :result
 :ok? true
 :agent "name"
 :text "final reply"
 :raw {...}}
```

Composite nodes add more fields:

| Node | Extra fields |
| --- | --- |
| `pipe` | last child result fields. |
| `map` branches | `:results`, `:text`. |
| `map` planner/worker | `:subtasks`, `:results`, `:text`. |
| `reduce` | `:mapped`, `:reduced`, reducer `:text`. |
| `iterate` | `:rounds`, `:accepted?`. |
| `bind` routes | selected child result fields, or `:error :no-route`. |
