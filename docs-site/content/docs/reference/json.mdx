---
title: CLI
description: Run agents and workflow files from the karcarthy command.
---

Build the standalone CLI once:

```bash
clojure -T:build uber
```

Then use `bin/karcarthy` as a normal command:

```bash
./bin/karcarthy agent echo --instructions "Echo the input." hi
./bin/karcarthy run examples/workflows/echo.json hi
./bin/karcarthy demo
./bin/karcarthy --help
```

During source checkout development, the launcher can run from source if the
standalone jar is not built. For installed copies, set
`KARCARTHY_JAR=/path/to/karcarthy-0.0.2-standalone.jar`.

## Commands

### agent

Run one agent without writing a workflow file:

```bash
./bin/karcarthy agent reviewer \
  --instructions "Find the launch risk and answer in one sentence." \
  "Enterprise SSO is ready for staged beta."
```

By default, command output is just the result text. Add `--json` when a caller
needs the full result object:

```bash
./bin/karcarthy agent echo --instructions "Echo the input." hi --json
```

Useful options:

- `--adapter mock|claude`
- `--model NAME`
- `--tool NAME`
- `--mock-response AGENT=TEXT`
- `--input TEXT`

### run

Run a workflow JSON file. The file can be either a raw workflow object:

```json
{
  "type": "agent",
  "name": "echo",
  "instructions": "Echo the input."
}
```

or a full request object with `workflow`, `input`, and adapter fields.

```bash
./bin/karcarthy run examples/workflows/echo.json hi
./bin/karcarthy run examples/workflows/echo.json --input "hi" --json
```

### json

Use `json` for the machine protocol: request JSON on stdin, result JSON on
stdout.

```bash
./bin/karcarthy json < request.json
```

Top-level request shape:

```json
{
  "workflow": { "type": "agent", "name": "echo", "instructions": "Echo." },
  "input": "hello",
  "adapter": "mock",
  "mock-responses": {
    "echo": "hello back"
  }
}
```

`adapter` is `mock` by default. `claude` uses the Claude CLI adapter with
sub-agent defaults.

`mock-responses` is optional. When present with the mock adapter, it maps agent
names to deterministic response text. It is useful for examples and tests that
should show realistic output without calling a live model. Agents not listed
fall back to the default echo-style mock response.

`tools` is an adapter allowlist. karcarthy passes these names to adapters that
support tool selection; it does not create tool servers or bind MCP
configuration. The mock adapter ignores tools.

## JSON workflow shapes

### agent

```json
{
  "type": "agent",
  "name": "reviewer",
  "instructions": "Find launch risks.",
  "model": "sonnet",
  "tools": ["mcp__feedback__search"],
  "adapter": "claude"
}
```

### pipe

```json
{
  "type": "pipe",
  "steps": [
    { "type": "agent", "name": "researcher", "instructions": "Research." },
    { "type": "agent", "name": "writer", "instructions": "Write." }
  ]
}
```

### map

Branches on the same input:

```json
{
  "type": "map",
  "branches": [
    { "type": "agent", "name": "api", "instructions": "Review API." },
    { "type": "agent", "name": "docs", "instructions": "Review docs." }
  ]
}
```

Planner/worker shape:

```json
{
  "type": "map",
  "planner": { "type": "agent", "name": "planner", "instructions": "List subtasks." },
  "worker": { "type": "agent", "name": "worker", "instructions": "Handle one subtask." }
}
```

JSON cannot safely transmit host functions. To synthesize mapped output from a
non-JVM language, put a synthesizer agent after `map` in a `pipe`.

### iterate

```json
{
  "type": "iterate",
  "worker": { "type": "agent", "name": "writer", "instructions": "Draft." },
  "evaluator": { "type": "agent", "name": "critic", "instructions": "Reply ACCEPT or feedback." },
  "max-rounds": 3
}
```

### bind

Choice map:

```json
{
  "type": "bind",
  "source": { "type": "agent", "name": "classifier", "instructions": "Classify." },
  "routes": {
    "launch": { "type": "agent", "name": "launch", "instructions": "Handle launch." },
    "incident": { "type": "agent", "name": "incident", "instructions": "Handle incident." }
  },
  "default": { "type": "agent", "name": "fallback", "instructions": "Handle fallback." }
}
```

Continuation:

```json
{
  "type": "bind",
  "source": { "type": "agent", "name": "triage", "instructions": "Triage." },
  "to": { "type": "agent", "name": "specialist", "instructions": "Continue." }
}
```

## JSON response

`karcarthy json` and command runs with `--json` return the karcarthy result map
encoded as JSON:

```json
{
  "karcarthy/type": "result",
  "ok?": true,
  "agent": "writer",
  "text": "..."
}
```

If a JSON request fails to parse or run, the JSON protocol returns an object
with `ok: false`, `error`, and `exception`.
