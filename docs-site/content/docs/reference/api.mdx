Use the facade namespace for normal application code:

```clojure
(require '[karcarthy :as k])
```

## Agents

### `agent`

```clojure
(k/agent name instructions & {:keys [model tools adapter]})
```

Builds an agent map.

| Parameter | Type | Required | Meaning |
| --- | --- | --- | --- |
| `name` | string | yes | Stable agent name used in results, logs, and mock responses. |
| `instructions` | string | yes | Role/system instructions passed to the adapter. |
| `:model` | string | no | Adapter-specific model selector. |
| `:tools` | vector of strings | no | Adapter tool allowlist. Tools must already exist in the selected adapter environment. |
| `:adapter` | keyword | no | Adapter registry key used by `run`. |

Returns:

```clojure
{:karcarthy/type :agent
 :name "reviewer"
 :instructions "Find launch risks."
 :model "sonnet"
 :tools ["mcp__feedback__search"]
 :adapter :claude}
```

Example:

```clojure
(k/agent "reviewer"
  "Find launch risks and propose mitigations."
  :model "sonnet"
  :tools ["mcp__feedback__search"])
```

### `defagent`

```clojure
(k/defagent symbol instructions & opts)
```

Defines a var containing an agent. The symbol name becomes the agent `:name`.

```clojure
(k/defagent researcher
  "Research the question and cite sources."
  :model "sonnet")
```

### `agent?` and `explain-agent`

```clojure
(k/agent? value)
(k/explain-agent value)
```

`agent?` returns true for well-formed agent maps. `explain-agent` returns a
human-readable validation message or nil.

## Workflows

### `pipe`

```clojure
(k/pipe & steps)
```

Runs steps in sequence. Each step receives the previous result's `:text`.

```clojure
(k/pipe researcher summarizer)
```

### `map`

```clojure
(k/map workflows & {:keys [max-concurrency]})
(k/map planner worker & {:keys [max-concurrency]})
```

With a workflow collection, runs every branch on the same input. With planner
and worker, reads planner EDN `{:subtasks [...]}` and runs the worker for each
subtask.

```clojure
(k/map [product-reviewer engineering-reviewer security-reviewer]
       :max-concurrency 4)

(k/map planner worker :max-concurrency 8)
```

### `reduce`

```clojure
(k/reduce mapped reducer & {:keys [max-concurrency]})
```

Runs mapped work, then sends an EDN result summary to a reducer workflow.

```clojure
(k/reduce
  (k/map [product-reviewer engineering-reviewer])
  brief-writer)
```

The reducer receives EDN shaped like `{:input ... :subtasks ... :results [...]}`.

### `iterate`

```clojure
(k/iterate worker evaluator & {:keys [max-rounds]})
```

Runs a worker/evaluator loop until accepted or the round limit is reached.

```clojure
(k/iterate draft-writer critic :max-rounds 3)
```

Evaluator workflows reply with EDN verdict maps:

```clojure
{:accept? true}
{:accept? false :feedback "specific revision feedback"}
```

### `bind`

```clojure
(k/bind source routes & {:keys [default]})
(k/bind source continuation & {:keys [prompt]})
```

With a route map, `source` replies with EDN `{:route key}` and the selected
workflow receives the original input.

```clojure
(k/bind classifier
        {:launch launch-brief
         :incident incident-plan}
        :default launch-brief)
```

With two workflows, the continuation receives the source result text.

```clojure
(k/bind triage-agent specialist-agent)
```

### `workflow?` and `defworkflow`

```clojure
(k/workflow? value)
(k/defworkflow symbol workflow)
```

Use these when storing, generating, or publishing workflows from application
code.

## Schemas

### `edn-schema` and `json-schema`

```clojure
k/edn-schema
k/json-schema
```

These are plain data reference values for the public workflow shapes. Use
`edn-schema` when documenting or generating EDN workflows, and `json-schema`
for the CLI JSON bridge.

```clojure
(keys k/edn-schema)
;=> (:agent :result :workflow :pipe :map-branches ...)

(get k/json-schema "$ref")
;=> "#/$defs/workflow"
```

## Rewrites

Workflow rewrites operate on EDN before execution. They never call adapters and
never use `eval`.

### `agents`

```clojure
(k/agents workflow)
```

Returns the agent leaves in traversal order.

### `over`

```clojure
(k/over pred f workflow)
```

Applies `f` to every value in `workflow` where `pred` returns true, then
validates the rewritten workflow.

```clojure
(k/over k/agent?
        #(update % :instructions str "\n\nState assumptions before final answer.")
        workflow)
```

### `config`

```clojure
(k/config opts workflow)
```

Applies common agent configuration in one pass. Supported keys:

| Key | Meaning |
| --- | --- |
| `:adapter` | Adapter registry key. |
| `:model` | Model selector string. |
| `:instructions/suffix` | Text appended to every agent instruction. |

```clojure
(def production-workflow
  (->> workflow
       (k/config {:adapter :claude
                  :model "claude-sonnet-4"
                  :instructions/suffix "State assumptions before final answer."})))
```

## Execution

### `run`

```clojure
(k/run adapter workflow input)
(k/run adapter workflow input opts)
```

Interprets workflow data through an adapter or adapter registry.

```clojure
(k/run (k/mock-adapter)
       (k/pipe researcher summarizer)
       "map the agent framework space")
```

Returns a result map:

```clojure
{:karcarthy/type :result
 :ok? true
 :agent "summarizer"
 :text "..."}
```

Top-level `run` throws for malformed input. Composite nodes convert child
exceptions into not-ok result maps.

## Adapters

### `mock-adapter`

```clojure
(k/mock-adapter)
(k/mock-adapter respond-fn)
```

Offline deterministic adapter for tests and examples.

```clojure
(k/mock-adapter
 (fn [{:keys [agent prompt]}]
   (str "[" (:name agent) "] " prompt)))
```

### `claude-cli`

```clojure
(k/claude-cli)
(k/claude-cli default-opts)
```

Runs agent calls through the Claude CLI. Common options include model, working
directory, turn limits, and extra Claude CLI args.

```clojure
(k/claude-cli {:model "haiku"
               :max-turns 4})
```

### `command-adapter`

```clojure
(k/command-adapter argv-or-fn)
(k/command-adapter argv-or-fn opts)
```

Wraps a local command. The command receives the prompt on stdin and stdout
becomes result text.

```clojure
(k/command-adapter ["tr" "a-z" "A-Z"])
(k/command-adapter (fn [agent] ["ollama" "run" (:model agent)])
                   {:timeout-ms 120000})
```

Options:

| Option | Meaning |
| --- | --- |
| `:trim?` | Trim stdout; defaults to true. |
| `:env` | Extra environment variables. |
| `:dir` | Working directory. |
| `:timeout-ms` | Kill the command after this many milliseconds. |

### `openai-agents-sdk`

```clojure
(k/openai-agents-sdk)
(k/openai-agents-sdk default-opts)
```

Delegates single-agent calls to the OpenAI Agents SDK bridge.

```clojure
(k/openai-agents-sdk {:python-bin "python3"
                      :timeout-ms 120000})
```

### Adapter registries

```clojure
(def adapters
  {:default (k/mock-adapter)
   :claude  (k/claude-cli {})})

(k/run adapters
       (k/pipe
        (k/agent "planner" "Plan." :adapter :claude)
        (k/agent "checker" "Check." :adapter :default))
       "ship the docs")
```

## Parsing generated data

```clojure
(k/read-workflow text)
(k/read-agent text)
(k/dsl-reference)
```

`read-workflow` and `read-agent` parse EDN emitted by an agent without using
`eval`.

## Results

```clojure
(k/result map)
(k/ok? result)
```

`result` fills in the standard result tag and defaults `:ok?` to true.

```clojure
(k/result {:agent "reviewer" :text "ready"})
;=> {:karcarthy/type :result
;    :ok? true
;    :agent "reviewer"
;    :text "ready"}
```

Composite workflows may add `:results`, `:subtasks`, `:rounds`, `:accepted?`,
or `:label`.
