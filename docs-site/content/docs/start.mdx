## Requirements

- JDK 21 or newer
- Clojure CLI
- A git SHA from `soohoonc/karcarthy`

karcarthy has no published release yet, so depend on it from git:

```clojure title="deps.edn"
{:deps {io.github.soohoonc/karcarthy
        {:git/url "https://github.com/soohoonc/karcarthy"
         :git/sha "<commit sha>"}}}
```

## Define agents

Use the facade namespace for the common API:

```clojure
(require '[karcarthy :as k])

(k/defagent researcher
  "Research the question and cite sources.")

(k/defagent summarizer
  "Summarize the findings in one sentence.")
```

Agents are just maps:

```clojure
(k/agent "planner"
  "Break the task into steps."
  :model "sonnet"
  :tools ["mcp__docs__search" "mcp__repo__read"])
```

`tools` is an adapter allowlist. Those names must already be configured in the
selected Agent SDK, CLI, or MCP layer; karcarthy only passes the allowlist
through.

## Run offline first

The mock adapter is deterministic and does not call a model:

```clojure
(k/run (k/mock-adapter)
       (k/pipe researcher summarizer)
       "what is a monad?")
```

The result has one shape across adapters:

```clojure
{:karcarthy/type :result
 :ok? true
 :agent "summarizer"
 :text "..."}
```

## Swap in a live agent system

Once the workflow behaves correctly offline, swap in a live agent system:

```clojure
(k/run (k/claude-cli {})
       (k/pipe researcher summarizer)
       "what is a monad?")
```

The workflow data does not change. Only the adapter changes.

## Run as a command

Build the standalone runner:

```bash
clojure -T:build uber
```

Then run agents or workflow files:

```bash
./bin/karcarthy agent echo --instructions "Echo the input." hi
./bin/karcarthy run examples/workflows/echo.json hi
./bin/karcarthy demo
```

Python, TypeScript, and other non-JVM hosts use this command path. The command
still runs the same workflow data and adapter protocol as the Clojure API.

## Run the local checks

From the repository root:

```bash
clojure -M:test
clojure -M -m karcarthy.demo
clojure -T:build uber
./bin/karcarthy agent echo --instructions "Echo the input." hi
```

The demos and tests default to offline adapters unless a file explicitly calls
out a live model dependency.
