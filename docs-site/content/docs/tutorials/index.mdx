These tutorials build the same launch-readiness workflow in three languages.
The workflow classifies a request, runs specialist reviewers, combines their
notes, drafts a brief, and loops through critique.

The workflow is intentionally more than a toy, and the agents are intentionally
more than one-line prompts.

## Agent configuration

Each language builds agents from profile data before turning them into ordinary
karcarthy agent maps. The profile fields are application UI, not new workflow
resource types:

- role and mission define identity.
- operating context acts like a compact workspace bootstrap.
- tools are an adapter allowlist, not provisioned infrastructure.
- responsibilities describe the job to do.
- output contract makes downstream composition predictable.
- tone controls the user-facing experience.
- boundaries and self-checks keep the agent honest.

The product owns the configuration screen and templates. karcarthy only needs
the final agent instructions and workflow data.

Tool names must already exist in the selected Agent SDK, CLI, or MCP
configuration. karcarthy passes the names through as an allowlist for adapters
that support it, such as Claude CLI `--allowedTools`. It does not create MCP
servers, databases, or tool implementations. The offline mock adapter ignores
tools entirely; the examples include tool names only to show the shape a real
host application would pass after configuring those tools.

## Workflow shape

The workflow uses the small functional vocabulary:

- `bind` chooses the launch path from a classifier result.
- `map` runs independent reviewers.
- `reduce` combines mapped reviewer output in Clojure.
- `pipe` turns review notes into a final draft.
- `iterate` gives the draft one or more critique rounds.

Python and TypeScript send JSON to `bin/karcarthy`, so host-language functions
cannot be sent as reducers. Those guides use a synthesizer agent in a `pipe`
after `map`, which is the portable cross-language shape. Clojure can also use a
host function with `reduce`.

## Run the examples

From the repository root:

```bash
clojure -M -e '(load-file "examples/clojure/launch.clj")'
python3 examples/python/launch.py
npx tsx examples/typescript/launch.ts
```

The examples default to the offline mock adapter so you can see the workflow
shape without making paid model calls. Swap the adapter to `claude` where noted
to run the same orchestration through the Claude CLI.
