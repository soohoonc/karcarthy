---
title: Workflows
description: Compose agents with pipe, map, reduce, iterate, and bind.
---

Workflows are maps tagged with `:karcarthy/type`. Builder functions make them
pleasant to write, but the result is still plain EDN. The public names are the
small functional set: `pipe`, `map`, `reduce`, `iterate`, and `bind`.

## pipe

`pipe` runs steps in order, passing each result text to the next step.

```clojure
(k/run agent-system
       (k/pipe researcher summarizer)
       "map the agent framework space")
```

## map

`map` applies work across a collection. Pass a collection of workflows to run
them all on the same input:

```clojure
(k/map [api-reviewer docs-reviewer ux-reviewer])
```

Pass a planner and worker when the planner should produce subtasks and the
worker should run once per subtask:

```clojure
(k/map planner writer)
```

## reduce

`reduce` combines the results of mapped work.

```clojure
(k/reduce summarize-results
          (k/map [api-reviewer docs-reviewer ux-reviewer]))
```

For planner/worker mapping, the reducer receives `[results input]`.

## iterate

`iterate` loops a worker through a reviewer until a stop condition or round
limit is reached.

```clojure
(k/iterate draft-writer critic :max-rounds 3)
```

## bind

`bind` uses one workflow's result to select or continue to the next workflow.
With a choice map, the first workflow chooses the key and the selected workflow
receives the original input:

```clojure
(k/bind classifier
        {"bug" triage-bug
         "docs" triage-docs}
        :default fallback)
```

With two workflows, `bind` continues from the first result text into the second
workflow:

```clojure
(k/bind triage-agent specialist-agent)
```

## Data stays inspectable

Because every workflow is a value, you can transform it before running:

```clojure
(-> (k/pipe researcher summarizer)
    (update :steps conj reviewer)
    (->> (k/run agent-system "summarize the prior work")))
```

The important boundary is that karcarthy coordinates the graph. Agent SDKs,
coding agents, and CLIs still own the model call, tool call, and
provider-specific inner loop.
