---
title: Build Workflows
description: Understand how pipe, map, reduce, iterate, and bind move data through agents.
---

A workflow is a value. `run` interprets that value through an adapter and
returns a result map.

```clojure
(k/run adapter workflow "input text")
;=> {:karcarthy/type :result
;    :ok? true
;    :text "..."}
```

Every workflow node consumes text and returns a result. Composite nodes decide
how child result text is routed to the next node.

## Mental model

| Node | Use it when | Input behavior | Output behavior |
| --- | --- | --- | --- |
| `pipe` | Steps must happen in order. | First step receives the original input; each later step receives previous `:text`. | Last successful step's result. |
| `map` | Several reviewers or workers should run independently. | Each branch receives the same input, or each worker receives one planner subtask. | A result with `:results` and combined `:text`. |
| `reduce` | Mapped outputs need deterministic synthesis. | Receives mapped child results in host code. | Whatever result your reducer returns. |
| `iterate` | A draft needs critique and revision. | Worker receives the input, then revision prompts. | Final draft plus `:rounds` and `:accepted?`. |
| `bind` | A classifier should choose the next path, or one step should continue into another. | Source receives original input. | Selected/continued workflow result. |

## pipe

Use `pipe` for sequential work:

```clojure
(def workflow
  (k/pipe researcher summarizer))

(k/run (k/mock-adapter) workflow "what is a monad?")
```

Trace with the mock adapter:

```text
input:      what is a monad?
researcher: [researcher] what is a monad?
summarizer: [summarizer] [researcher] what is a monad?
```

The next step receives only the previous result text. If a step fails, the pipe
returns that not-ok result and stops.

## map

Use `map` when independent agents should inspect the same input:

```clojure
(def reviewers
  (k/map [product-reviewer engineering-reviewer security-reviewer]))
```

Trace:

```text
input: enterprise SSO beta
product-reviewer:     reviews the same input
engineering-reviewer: reviews the same input
security-reviewer:    reviews the same input
```

The result contains ordered child results:

```clojure
{:karcarthy/type :result
 :ok? true
 :results [{:agent "product-reviewer" :text "..."}
           {:agent "engineering-reviewer" :text "..."}
           {:agent "security-reviewer" :text "..."}]
 :text "...joined child text..."}
```

Use planner/worker mapping when a planner should create subtasks:

```clojure
(k/map planner worker :max-concurrency 8)
```

The planner's text is parsed as one non-blank subtask per line. The worker runs
once per subtask.

## reduce

Use `reduce` when mapped outputs need deterministic host-code synthesis:

```clojure
(defn combine-review-notes
  ([results] (combine-review-notes results nil))
  ([results _input]
   (k/result
    {:text (str "Review packet:\n"
                (str/join "\n" (map #(str "- " (:text %)) results)))})))

(def review-packet
  (k/reduce combine-review-notes
            (k/map [product-reviewer engineering-reviewer security-reviewer])))
```

Reducers are Clojure functions. Python and TypeScript cannot send functions
through JSON, so those examples place a synthesizer agent after `map` inside a
`pipe`.

## iterate

Use `iterate` for draft, critique, revise loops:

```clojure
(def launch-brief
  (k/iterate brief-writer critic :max-rounds 2))
```

Trace:

```text
round 1 worker:    writes a draft
round 1 evaluator: replies ACCEPT or feedback
round 2 worker:    receives input, previous draft, and feedback
```

The result includes loop metadata:

```clojure
{:karcarthy/type :result
 :ok? true
 :text "final draft"
 :rounds 2
 :accepted? true}
```

Evaluator workflows accept when their text starts with `ACCEPT`.

## bind

Use `bind` with a route map when one workflow chooses the next path:

```clojure
(k/bind classifier
        {"launch" launch-brief
         "incident" incident-plan}
        :default launch-brief)
```

Trace:

```text
input:     Prepare a launch review.
classifier result text: launch
selected workflow:      launch-brief
launch-brief input:     Prepare a launch review.
```

Use `bind` with two workflows for continuation:

```clojure
(k/bind triage-agent specialist-agent)
```

In continuation form, the second workflow receives the source result text.

## Transform before running

Because workflows are data, application code can inspect and rewrite them:

```clojure
(-> (k/pipe researcher summarizer)
    (update :steps conj reviewer)
    (->> (k/run (k/mock-adapter) "summarize the prior work")))
```

That is the point of karcarthy: keep the coordination graph visible and
serializable while adapters own model calls, tools, and provider-specific agent
loops.
