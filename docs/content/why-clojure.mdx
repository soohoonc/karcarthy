---
title: Why Clojure?
description: How forms, macros, and evaluation let an Agent add new Agents while it runs.
---

karcarthy depends on a few properties that Clojure makes ordinary: programs
can be read as data, macros receive their source forms, and evaluated forms
become running code.

The harness uses those properties to let an Agent add new Agents while it runs.

## `agent` is a macro

An Agent begins with a macro invocation:

```clojure
(def reviewer
  (k/agent
   {:name "reviewer"
    :model {:transport :responses :id "gpt-5.6"}
    :instructions "Review the proposed change."
    :input string?
    :output string?}))
```

The macro constructs an Agent value and retains both the original form and its
expansion:

```clojure
(k/definition reviewer)
(k/expansion reviewer)
```

The retained form shows exactly what was evaluated.

## A form can become a running Agent

During a Run, the model can call the built-in `agent` Tool:

```clojure
{:source
 "(agent
    {:name \"reviewer\"
     :model {:transport :responses :id \"gpt-5.6\"}
     :instructions \"Review the proposed change.\"
     :input string?
     :output string?})"
 :input
 "Replace synchronous writes with a queue."}
```

karcarthy processes the source as Clojure:

```text
read one form
  → expand the agent macro
  → check the expansion
  → evaluate the form
  → verify the Agent value
  → run the Agent
```

The source must contain exactly one top-level `agent` form. The resulting Agent
runs inside the current Run, and its output returns to the parent as the Tool
result.

## The program can extend itself

A generated Agent is another Agent:

```clojure
(agent
 {:name "researcher"
  :model {:transport :responses :id "gpt-5.6"}
  :instructions "Research the assigned question."
  :input string?
  :output string?})
```

The new Agent receives the built-in `agent` Tool and can create another one.
The running program gains more behavior by evaluating more Clojure.

## Control flow stays in Clojure

Known coordination does not need another representation:

```clojure
(if high-risk?
  (k/run! security-reviewer change)
  (k/run! general-reviewer change))
```

```clojure
(loop [attempt 1]
  (let [run (k/run! reviewer change)]
    (if (acceptable? run)
      run
      (recur (inc attempt)))))
```

karcarthy owns the model and Tool loop. Clojure owns branching, iteration,
recursion, functions, macros, and concurrency.
