---
title: Events and monitoring
description: Observe runs, Agent calls, model output, Tools, and eval.
---

Pass an event handler to `run!`:

```clojure
(def seen (atom []))
(def result
  (k/run! assistant input {:on-event #(swap! seen conj %)}))
```

Or use the built-in live monitor:

```clojure
(def live (k/monitor {:display :tree}))
(def result (k/run! assistant input {:on-event live}))
```

```text
Run run_7c2e9b… · running · 12s · 4 model calls · 4,218 tokens · 1 eval
└─ architect · waiting for Agents
   ├─ botanist · calling model
   └─ radiation-engineer · calling model
```

`(k/monitor live)` prints the latest view again. `(k/monitor-state live)`
returns the event-derived data. One monitor may observe several runs.

## Event families

```clojure
:run/started          :run/completed       :run/failed
:run/cancelled
:agent/started        :agent/completed     :agent/failed
:model/requested      :model/text-delta    :model/completed
:model/failed
:tool/started         :tool/completed      :tool/failed
:session/updated
:eval/started         :eval/expanded       :eval/completed
:eval/failed
```

All events carry `:run-id` and time. Agent-scoped events carry `:agent-id`,
`:parent-id`, and `:depth`. Calls to `run!` that participate in an existing run
appear as Agent branches under the same run ID; they do not emit another Run
lifecycle.

Eval's expanded event retains both the parsed expression and macroexpansion:

```clojure
(->> (k/events result)
     (filter #(= :eval/expanded (:type %)))
     (map #(select-keys % [:expression :expansion :namespace])))
```

## Application events

Tools and evaluated Clojure may emit domain events:

```clojure
(k/emit! {:type :application/decision
          :decision :retry})
```

karcarthy adds run and Agent lineage before storing and publishing the event.
See the [event catalog](/docs/reference/events) for the stable names.
