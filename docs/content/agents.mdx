An Agent is a Clojure value containing a model, instructions, capabilities,
and boundary contracts.

```clojure
(k/defagent researcher
  {:description "Research one question and return sourced findings."
   :model {:transport :responses :id "gpt-5.6"}
   :instructions "Research carefully. Distinguish evidence from inference."
   :tools [web-search]
   :input string?
   :output string?})
```

When it runs, karcarthy validates the input, calls the model, executes requested
Tools or available Agents, and repeats until the model answers or the Run
stops.

## `agent` and `defagent`

`defagent` defines a var and derives the Agent name from its symbol:

```clojure
(k/defagent reviewer
  {:model {:transport :responses :id "gpt-5.6"}
   :instructions "Review the change."
   :output string?})
```

Use `agent` inside an expression and supply `:name` explicitly:

```clojure
(def reviewer
  (k/agent
   {:name "reviewer"
    :model {:transport :responses :id "gpt-5.6"}
    :instructions "Review the change."
    :output string?}))
```

Both macros retain their original form and macroexpansion:

```clojure
(k/definition reviewer)
(k/expansion reviewer)
```

## Runs

`run!` passes one input value to an Agent and returns a Run:

```clojure
(def run
  (k/run! researcher "What changed in the latest Java release?"))

(select-keys run [:status :output :usage :error])
```

A Run is `:completed`, `:failed`, or `:cancelled`. Completion means the Agent
and its contracts completed successfully; it does not mean an external
evaluator judged the answer correct. Check `:status` before consuming
`:output`.

Each top-level `run!` creates a bounded execution with its own limits, events,
and optional Session. An Agent called through `:agents` or created through the
built-in `agent` Tool remains nested in that Run.

## Make a specialist available

Put existing Agents in `:agents` when the parent model should decide whether
to call them:

```clojure
(k/defagent support
  {:description "Resolve one billing question."
   :model {:transport :responses :id "gpt-5.6"}
   :instructions "Resolve the assigned billing question."
   :output string?})

(k/defagent manager
  {:model {:transport :responses :id "gpt-5.6"}
   :instructions "Resolve the request. Ask support when useful."
   :agents [support]
   :output string?})
```

The specialist receives only the explicit call input, not the parent's
conversation. Its output returns to the parent model. Limits, usage,
cancellation, approvals, and events remain in the same Run.

## Compose Agents in Clojure

When application code should decide ordering or concurrency, use an ordinary
function:

```clojure
(defn review-change [change]
  (->> [security-reviewer api-reviewer]
       (mapv #(future (k/run! % change)))
       (mapv deref)
       (mapv :output)))
```

The [composition example](/docs/guides/composition) handles failed Runs and
passes both reviews to an editor.

## Generate an Agent at runtime

Every Agent also receives a built-in Tool named `agent`. The model calls it
with one Clojure Agent form and explicit input:

```clojure
{:source "(agent {:name \"specialist\" ...})"
 :input "Complete task input"}
```

karcarthy evaluates the form, runs the resulting Agent inside the current Run,
and returns its output to the parent model. [Generated
agents](/docs/generated-agents) explains the mechanism; [Create agents at
runtime](/docs/guides/architect) shows it running.

## Configuration

| Key | Meaning |
| --- | --- |
| `:name` | Stable runtime name. `agent` requires it; `defagent` derives it. |
| `:description` | Tells a parent model when to call this Agent. |
| `:model` | Selects the model and transport. |
| `:instructions` | Agent-specific instructions appended to the base prompt. |
| `:tools` | Tools the model may call. |
| `:agents` | Existing Agents the model may call. |
| `:input` / `:output` | Contracts checked at the Agent boundary. |
| `:input-schema` / `:output-schema` | Optional JSON schemas shown to the model. |
| `:limits` | Limits that narrow the containing Run for this Agent. |

Pass local dependencies to Tools through Run `:context`:

```clojure
(k/run! account-agent "Summarize account 123."
        {:context {:account-id "123" :db db}
         :limits {:model-calls 40
                  :depth 4
                  :parallelism 4
                  :agent-forms 8
                  :deadline-ms 300000}})
```

The [API reference](/docs/reference/api#agent-and-defagent) lists every option.
