An Agent is a flat Clojure map that describes a model, its instructions, the
Tools and Agents it may call, and its input and output boundaries.

```clojure
(require '[karcarthy :as k])

(k/defagent researcher
  {:description "Research one question and return sourced findings."
   :model "gpt-5.6"
   :instructions "Research carefully. Distinguish evidence from inference."
   :tools [web-search]})
```

`defagent` derives `:name` from the var. Use `agent` to construct a value inside
an expression and provide `:name` explicitly:

```clojure
(def researcher
  (k/agent
   {:name "researcher"
    :model "gpt-5.6"
    :instructions "Research carefully."}))
```

## Configuration

| Key | Meaning |
| --- | --- |
| `:name` | Stable Agent name. `agent` requires it; `defagent` derives it. |
| `:description` | Tells another model when to call this Agent. |
| `:model` | Model ID, model configuration, or context function. |
| `:instructions` | Complete model-visible string or context function. |
| `:tools` | Tools available to the model; include `k/eval` for dynamic Clojure orchestration. |
| `:agents` | Agents available through delegation. |
| `:context-schema` | Schema for local Run context. |
| `:input-schema` / `:output-schema` | Agent input and output boundaries; each defaults to `string?`. |
| `:input-guardrails` / `:output-guardrails` | Functions that check input and output. |
| `:max-turns` | Maximum model turns for this Agent call. |
| `:limits` | Limits that may narrow the active Run. |

Instructions are exact: karcarthy does not prepend a framework prompt. The
`:model`, `:instructions`, `:tools`, and `:agents` values may be functions of
the current Run context when configuration needs local dependencies.

See [Models](/docs/protocols/models) for transport configuration and [Context](/docs/context)
for context functions.

## Schemas and guardrails

Schemas validate the Agent boundary. Input is checked before the first model
request; output is checked before the Agent completes.

Guardrails run in the same positions and receive the local Run context plus the
value:

```clojure
{:input-guardrails
 [(fn [run-context value]
    (allowed-request? (:context run-context) value))]
 :output-guardrails
 [(fn [_ value]
    (grounded? value))]}
```

A schema or guardrail failure produces a structured failed Run.

## Agents are data

Normal Clojure operations work directly on an Agent:

```clojure
(:instructions researcher)
(assoc researcher :instructions "Return exactly three findings.")
```

The construction macros retain their source definition and macroexpansion for
inspection:

```clojure
(k/definition researcher)
(k/expansion researcher)
```

There is no hidden configuration object or separate workflow representation.
