title: Agents and runs
description: Define Agent data and execute it with run!.
---

An Agent is a flat Clojure map containing a model, instructions, Tools, other
Agents it may call, and boundary contracts.

```clojure
(k/defagent researcher
  {:description "Research one question and return sourced findings."
   :model "gpt-5.6"
   :instructions "Research carefully. Distinguish evidence from inference."
   :tools [web-search]
   :input string?
   :output string?})
```

`defagent` derives `:name` from the var. Use `agent` for an expression and pass
`:name` explicitly. Both macros retain their definition and expansion:

```clojure
(k/definition researcher)
(k/expansion researcher)
(assoc researcher :instructions "Return three findings.")
```

## Models

A string selects the OpenAI Responses transport:

```clojure
{:model "gpt-5.6"}
```

Use a map for advanced configuration:

```clojure
{:model {:transport :responses
         :provider :openai
         :id "gpt-5.6"
         :reasoning :medium
         :timeout-ms 180000}}
```

## Runs

```clojure
(def result (k/run! researcher "What changed in Java?"))
(select-keys result [:status :output :usage :error])
```

A Run is `:completed`, `:failed`, or `:cancelled`. Invalid configuration throws
before execution. Model, Tool, contract, budget, deadline, and cancellation
failures during execution produce a failed or cancelled Run map.

The first `run!` establishes a run. Calls within its dynamic extent participate
in that run and share limits, usage, events, context, approvals, cancellation,
and its ID. Each Agent starts a fresh model conversation. Only the first Agent
uses the optional Session.

## Existing Agents as Tools

Put Agents in `:agents` when the model should decide whether to call them:

```clojure
(k/defagent manager
  {:model "gpt-5.6"
   :instructions "Resolve the request. Ask support when useful."
   :agents [support]
   :output string?})
```

The called Agent receives only its explicit input, not the parent's model
conversation. Its output returns as the Tool result.

## Dynamic composition

Every Agent also receives `eval`. The model may evaluate one Clojure expression
that creates and runs Agents:

```clojure
{:code "(let [a (agent {:name \"specialist\" ...})]
          (:output (run! a input)))"
 :input "Complete task input"}
```

See [Dynamic eval](/docs/eval).

## Configuration

| Key | Meaning |
| --- | --- |
| `:name` | Stable name. `agent` requires it; `defagent` derives it. |
| `:description` | Tells another model when to call this Agent. |
| `:model` | Model ID string, model map, or metadata function. |
| `:instructions` | String or metadata function. |
| `:tools` / `:agents` | Tools and Agents available to the model. |
| `:context` | Contract for local run context. |
| `:input` / `:output` | Boundary contracts. |
| `:input-schema` / `:output-schema` | JSON schemas exposed to models. |
| `:max-turns` | Maximum model turns before the Agent fails. |
| `:guardrails` | Input and output checks. |
| `:limits` | Limits that can only narrow the active run. |
| `:metadata` | Application data. |

```clojure
(k/run! account-agent "Summarize account 123."
        {:context {:account-id "123" :db db}
         :limits {:model-calls 40
                  :depth 4
                  :parallelism 4
                  :evals 8
                  :deadline-ms 300000}})
```
