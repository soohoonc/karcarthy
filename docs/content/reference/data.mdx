## Agent

```clojure
{:karcarthy/type :agent
 :name "researcher"
 :model {:transport :responses :provider :openai :id "gpt-5.6"}
 :instructions "Research carefully."
 :tools [...]
 :input string?
 :output string?
 :definition-ns 'example.research
 :definition '(agent {...})
 :expansion '(...)}
```

Agent configuration is flat. `definition` and `expansion` are additional
inspection data, not a second configuration layer.

## Tool

```clojure
{:karcarthy/type :tool
 :name "lookup"
 :description "Look up a record."
 :input map?
 :input-schema {:type "object" ...}
 :output map?
 :definition '(tool {...} [input] ...)
 :expansion '(...)
 :execute #object[clojure.lang.IFn ...]}
```

If karcarthy cannot derive a meaningful JSON Schema from `:input`, the Tool
must provide `:input-schema`.

## Run

```clojure
{:karcarthy/type :run
 :id "run_..."
 :status :completed
 :agent "researcher"
 :input "question"
 :output "answer"
 :usage {:model-calls 3
         :input-tokens 12000
         :output-tokens 900
         :evals 1
         :duration-ms 4200.0}
 :events [...]
 :error nil}
```

Participating `run!` calls return the same shape and same `:id`. Their `:events`
cover the Agent call; the first Run retains the aggregate event sequence.

## Failure

```clojure
{:karcarthy/type :failure
 :kind :schema
 :phase :tool-output
 :message "Value does not satisfy schema for tool-output"
 :data {...}
 :recoverable? false}
```

## Monitor snapshot

`monitor-state` returns process-local data derived from events. Each run entry
contains status, elapsed time, usage, eval count, Agent order, parent IDs,
activities, active Tools, and errors. The completed Run and its events are the
record; the monitor is a projection for display.
