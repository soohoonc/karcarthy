---
title: Data
description: Agent, Tool, Run, Session, event, and failure values.
---

karcarthy values are ordinary Clojure data. Agent and Tool values also contain
functions, so applications choose how to project them into an external format.

## Agent

```clojure
{:karcarthy/type :agent
 :name "researcher"
 :config {...}
 :definition-ns 'example.research
 :definition '(agent {...})
 :expansion '(...)
 :body nil}
```

The public config keys are `:name`, `:description`, `:model`, `:instructions`,
`:context`, `:input`, `:input-schema`, `:tools`, `:agents`, `:output`,
`:output-schema`, `:max-turns`, `:stop-when`, `:guardrails`, `:limits`, and
`:metadata`.

`definition` and `expansion` are retained for program analysis. `body` is nil
for the normal model loop and a compiled function for a Clojure Agent program.

## Tool

```clojure
{:karcarthy/type :tool
 :name "web-search"
 :description "Search the web."
 :config {...}
 :definition '(tool {...} [input] ...)
 :expansion '(...)
 :execute #object[clojure.lang.IFn ...]}
```

Tool input is validated before execution. Tool output is validated before it
is returned to the model.

## Run

```clojure
{:karcarthy/type :run
 :id "run_..."
 :status :completed
 :agent "researcher"
 :input "question"
 :output "answer"
 :usage {:model-calls 3
         :input-tokens 12000
         :output-tokens 900
         :agent-forms 1
         :duration-ms 4200.0}
 :events [...]
 :error nil}
```

Status is `:completed`, `:failed`, or `:cancelled`. Completion means the
harness contracts passed; an external evaluator may still score the behavior
poorly.

## Run monitor snapshot

Dereferencing a Run monitor returns process-local data derived from observed
events:

```clojure
{:karcarthy/type :monitor-snapshot
 :run-order ["run_..."]
 :runs
 {"run_..."
  {:id "run_..."
   :status :running
   :agent "architect"
   :agent-forms 2
   :created-agents ["failure-analyst" "rollout-planner"]
   :agent-order ["agent_..." "agent_..." "agent_..."]
   :agents
   {"agent_..."
    {:id "agent_..."
     :parent-id nil
     :depth 0
     :name "architect"
     :status :running
     :activity :waiting-agents
     :tools {...}}}}}}
```

The monitor is a live display. The completed Run and its event vector are the
permanent record.

## Session

A Session implements `karcarthy.session/Session`:

```clojure
(session-id session)
(get-items session)
(add-items! session items)
(pop-item! session)
(clear-session! session)
```

It stores chronological conversation messages across Runs. It is not a Run
checkpoint or general application-state store.

## Events

Every event has `:type`, `:time-ms`, and `:run-id`. Agent events add
`:agent-id`, `:parent-id`, and `:depth`; Tool events may add `:tool-call-id`.

```clojure
:run/started          :run/completed       :run/failed
:run/cancelled
:agent/started        :agent/completed     :agent/failed
:model/requested      :model/text-delta    :model/completed
:model/failed
:tool/started         :tool/completed      :tool/failed
:session/updated
:program/read         :program/expanded    :program/checked
:program/evaluated    :program/failed
```

The `:program/*` events describe generated-Agent source compilation, not
ordinary Agent execution. See the [compiler event
table](/docs/reference/agent-forms#compiler-events) for their sequence and
fields.

## Failure

```clojure
{:karcarthy/type :failure
 :kind :contract
 :phase :tool-output
 :message "Value does not satisfy contract for tool-output"
 :data {...}
 :recoverable? false}
```

Failed and cancelled Runs retain their events and failure data but have no
output.
