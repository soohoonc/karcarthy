## `agent` and `defagent`

```clojure
(k/agent config)
(k/agent config [input] body ...)

(k/defagent name config)
(k/defagent name config [input] body ...)
```

Both forms create an Agent. `defagent` defines a var and supplies `:name` from
its symbol. `agent` creates a value and requires `:name` in the config.

Without a body, an Agent requires `:model` and `:instructions` and uses the
model/Tool loop. With `[input] body ...`, the Clojure program computes its
output directly.

```clojure
{:name "researcher"
 :description "Research one question."
 :model {:transport :responses :id "gpt-5.6"}
 :instructions "Research carefully."
 :context map?
 :input string?
 :input-schema {...}
 :tools [search-tool]
 :agents [fact-checker]
 :output ::report
 :output-schema {...}
 :max-turns 20
 :stop-when final-acceptable?
 :guardrails {:input [safe?] :output [grounded?]}
 :limits {:model-calls 30}
 :metadata {...}}
```

`:model`, `:instructions`, `:tools`, and `:agents` may be functions of the
current call metadata. Instructions must resolve to a string and are appended
to the packaged base system prompt automatically.

`:agents` makes those Agents callable by the model. An object-shaped
`:input-schema` or `:input` becomes the callable schema; otherwise the model
calls the Agent with `{"input": ...}`.

Every model Agent may submit a new Clojure `agent` form and run the resulting
Agent inside the current Run.

```clojure
(k/agent? value)
(k/definition agent)
(k/expansion agent)
```

`definition` and `expansion` expose the retained Clojure forms for tooling and
experiments. They are not needed to run an Agent.

## `tool` and `deftool`

```clojure
(k/tool config [input] body ...)
(k/deftool name config [input] body ...)
```

A Tool requires `:name`, `:description`, and `:input`; `deftool` supplies the
name. Optional keys are `:input-schema`, `:output`, `:output-schema`,
`:approval`, `:enabled?`, `:guardrails`, `:timeout-ms`, `:retry`,
`:to-model-output`, and `:metadata`.

```clojure
(k/tool? value)
```

## `run!`

```clojure
(k/run! agent input)
(k/run! agent input options)
```

Starts a Run and returns its complete result.

```clojure
{:context {...}
 :session session
 :limits {:model-calls 100
          :input-tokens 1000000
          :output-tokens 100000
          :depth 8
          :parallelism 6
          :agent-forms 20
          :deadline-ms 300000}
 :observe event-fn
 :approval approval-handler
 :cancel cancellation-token
 :metadata {...}
 :model-transports {...}
 :evaluation-namespace 'example.generated}
```

`context` returns the current Run's local dependency value from a Tool or
Clojure Agent program:

```clojure
(k/context)
```

## Sessions

```clojure
(def session (k/memory-session))
(def named (k/memory-session {:id "session_123" :items []}))

(k/session? session)
(k/session-id session)
(k/get-items session)
(k/add-items! session messages)
(k/pop-item! session)
(k/clear-session! session)
```

`memory-session` is thread-safe and process-local. Durable stores implement
`karcarthy.session/Session`.

## Prompts

```clojure
(k/prompt "Base instructions" (k/prompt-file "AGENTS.md"))
(k/system-prompt)
```

`prompt` joins strings and call-metadata functions. `prompt-file` reads UTF-8
text. `system-prompt` reads `resources/karcarthy/system.md`; karcarthy already
prepends that file to every model Agent, so applications normally use the
function only to inspect it or for explicit prompt composition.

## Agent forms

```clojure
(def form (k/read-agent-form source))
(def checked (k/check-agent-form! form))
(def generated (k/eval-agent-form! checked))

;; Read, expand, evaluate, and verify in one call:
(def generated (k/compile-agent! source))
```

These functions must run from an Agent program because compilation is charged
to the current Run. See [Agent forms](/docs/reference/agent-forms).

## Models

```clojure
{:transport :responses
 :provider :openai
 :id "gpt-5.6"}
```

The Responses transport supports `:base-url`, `:path`, `:url`, `:api-key`,
`:api-key-env`, `:headers`, `:auth?`, `:timeout-ms`, `:stream`, and
`:provider-options`. Direct OpenAI calls read `RESPONSES_API_KEY` or
`OPENAI_API_KEY` by default.

Use a deterministic model in tests:

```clojure
(k/fake-model (fn [request] {:type :final :output "done"}))
```

`model!` calls the active transport from a Clojure Agent program:

```clojure
(k/model! normalized-request)
```

## Supplied Tools

```clojure
(k/responses-web-search)
(k/local-tools {:cwd "/workspace"})

(def connection
  (k/connect-mcp! {:name "server"
                   :command "server-command"
                   :args ["--stdio"]}))
(def tools (k/mcp-tools connection))
(k/close-mcp! connection)
```

`local-tools` supplies `read`, `write`, `edit`, `bash`, and `search`. MCP uses
stdio and must be closed by the caller. See [Tools](/docs/tools),
[Models](/docs/models), and [MCP](/docs/protocols/mcp) for execution and lifecycle
semantics.

## ACP

```clojure
(k/serve-acp! {:agent assistant})
```

Or provide an Agent factory:

```clojure
(k/serve-acp!
 {:models ["gpt-5.6"]
  :agent (fn [{:keys [cwd model-id]}]
           (k/agent
            {:name "acp-agent"
             :model {:transport :responses :id model-id}
             :instructions "Complete the coding request."
             :tools (k/local-tools {:cwd cwd})
             :output string?}))})
```

Client-provided MCP Tools are appended automatically. See [ACP](/docs/protocols/acp) for
the complete session factory and supported protocol surface.

## Events and contracts

```clojure
(k/emit! {:type :application/decision :value value})
(k/events run)

(k/contract-valid? contract value)
(k/explain-contract contract value)
(k/contract->json-schema contract)
```
