---
title: Event catalog
description: Stable Run, Agent, model, Tool, Session, and program event names.
---

Every event contains `:karcarthy/type :event`, `:type`, `:time-ms`, and
`:run-id`. Agent-scoped events also contain `:agent-id`, `:parent-id`, and
`:depth`. Tool events add `:tool-call-id` where applicable.

## Runs

| Event | Emitted when |
| --- | --- |
| `:run/started` | A root Run begins. |
| `:run/completed` | The root Agent and output contract complete. |
| `:run/failed` | The Run ends with a structured failure. |
| `:run/cancelled` | Cancellation ends the Run. |

## Agents

| Event | Emitted when |
| --- | --- |
| `:agent/started` | An Agent begins inside the Run. |
| `:agent/completed` | Its output passes its contract. |
| `:agent/failed` | It ends with a failure. |

## Models

| Event | Emitted when |
| --- | --- |
| `:model/requested` | The harness sends a normalized model request. |
| `:model/text-delta` | Streaming model text arrives. |
| `:model/tool-call-delta` | Streaming Tool-call data arrives. |
| `:model/stream-event` | A transport exposes another streaming event. |
| `:model/completed` | A final response or Tool calls arrive; includes reported usage. |
| `:model/failed` | The model transport fails. |

## Tools

| Event | Emitted when |
| --- | --- |
| `:tool/approval-requested` | A Tool call requires a decision. |
| `:tool/approval-resolved` | The approval handler returns. |
| `:tool/approval-reused` | A prior once-per-Run approval applies. |
| `:tool/started` | Tool execution begins. |
| `:tool/completed` | Tool output passes its contract. |
| `:tool/failed` | Approval, execution, or output validation fails. |

## Sessions

| Event | Emitted when |
| --- | --- |
| `:session/updated` | A root model conversation is appended to its Session. |

## Generated programs

| Event | Emitted when |
| --- | --- |
| `:program/read` | Runtime Agent source enters the read phase. |
| `:program/expanded` | One `agent` form is read and macroexpanded. |
| `:program/checked` | The expansion passes structural checks. |
| `:program/evaluated` | Evaluation produces an Agent value. |
| `:program/failed` | A compiler phase fails; includes `:phase` and `:error`. |

Use `(k/events run)` for the retained vector or pass an observer through
`:observe` to consume events live. [Events and monitoring](/docs/events)
explains the execution model; [Run data](/docs/reference/data) documents the
surrounding values.
