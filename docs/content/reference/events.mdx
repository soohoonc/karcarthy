---
title: Event catalog
description: Stable Run, Agent, model, Tool, Session, and eval event names.
---

Every event contains `:karcarthy/type :event`, `:type`, `:time-ms`, and
`:run-id`. Agent-scoped events also contain `:agent-id`, `:parent-id`, and
`:depth`. Tool events add `:tool-call-id` where applicable.

| Family | Events |
| --- | --- |
| Run | `:run/started`, `:run/completed`, `:run/failed`, `:run/cancelled` |
| Agent | `:agent/started`, `:agent/completed`, `:agent/failed` |
| Model | `:model/requested`, `:model/text-delta`, `:model/tool-call-delta`, `:model/stream-event`, `:model/completed`, `:model/failed` |
| Tool | `:tool/approval-requested`, `:tool/approval-resolved`, `:tool/approval-reused`, `:tool/started`, `:tool/completed`, `:tool/failed` |
| Session | `:session/updated` |
| Eval | `:eval/started`, `:eval/expanded`, `:eval/completed`, `:eval/failed` |

Only the first `run!` emits Run lifecycle events. Participating Agent calls emit
Agent, model, Tool, and eval events under the same `:run-id`.

Use `(k/events result)` for retained lifecycle and application events, or pass
`:on-event` to consume every event live. Model text, Tool-call, and raw stream
deltas are live-only. Observer exceptions are ignored so telemetry cannot fail
the Agent Run. See [Eval reference](/docs/reference/eval) for eval event fields.
