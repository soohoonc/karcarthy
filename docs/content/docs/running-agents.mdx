---
title: Running agents
description: Run an Agent, inspect the result, and control context, limits, and cancellation.
---

`run!` is the application entry point:

```clojure
(def run
  (k/run! researcher "How does ACP model selection work?"))
```

For a model-backed Agent, the second argument is normally a user prompt. For a
Clojure-body Agent, it is the body's ordinary Clojure input value.

## Read the result

`run!` always returns a Run:

```clojure
{:karcarthy/type :run
 :id "run_..."
 :status :completed
 :agent "researcher"
 :input "How does ACP model selection work?"
 :output "..."
 :usage {:model-calls 4
         :input-tokens 12000
         :output-tokens 900
         :generated-forms 0
         :duration-ms 4200.0}
 :events [...]
 :error nil}
```

Status is `:completed`, `:failed`, or `:cancelled`. Check it before reading
`:output`. Failed and cancelled Runs retain usage, events, and a structured
`:error`.

## The model loop

For a model-backed Agent, karcarthy:

1. validates input and local context;
2. resolves the model, instructions, and Tools;
3. sends the prompt and conversation history to the model;
4. validates and executes Tool calls;
5. returns Tool results to the model; and
6. validates the final output.

The loop stops on a valid result, failure, cancellation, an Agent turn limit,
or a Run limit.

## Pass local context

Use `:context` for dependencies and request data that should not be inserted
into the prompt automatically:

```clojure
(k/run! account-agent "Summarize account 123."
        {:context {:account-id "123"
                   :db db}})
```

Agent bodies and Tools read it with `(k/context)`. An Agent's optional
`:context` contract validates the value.

## Compose Agents with Clojure

Every call to `run!` starts an independent Run with its own limits, events,
context, Session, and cancellation. Clojure-body Agents use the same function:

```clojure
(defn output! [run]
  (if (= :completed (:status run))
    (:output run)
    (throw (ex-info "Agent Run failed" {:run run}))))

(k/defagent review-team
  {:input string?
   :output vector?}
  [request]
  (->> [security-reviewer api-reviewer]
       (mapv #(future (k/run! % request)))
       (mapv deref)
       (mapv output!)))
```

Use normal Clojure for sequencing, branching, iteration, and concurrency.

This differs from an Agent entered through `as-tool` or the model-facing
`(agent)` Tool. Those Agents remain inside the parent's Run and share its
limits, usage, cancellation, approvals, and events.

## Set limits

```clojure
(k/run! agent prompt
        {:limits {:model-calls 100
                  :input-tokens 1000000
                  :output-tokens 100000
                  :agent-depth 8
                  :parallelism 6
                  :generated-forms 20
                  :deadline-ms 300000}})
```

`:agent-depth` and `:generated-forms` apply to Agents entered through Agent
Tools. `:parallelism` bounds concurrent Tool execution inside this Run; it does
not control ordinary Clojure `future` calls that start other Runs.

## Cancel a Run

Pass a Boolean, predicate, atom, or other dereferenceable value as `:cancel`:

```clojure
(def cancelled? (atom false))

(future
  (k/run! agent prompt {:cancel cancelled?}))

(reset! cancelled? true)
```

Cancellation is checked around Agent, model, and Tool calls.

## Run options

The remaining options are `:session`, `:observe`, `:approval`, `:metadata`,
`:model-transports`, and `:evaluation-namespace`. See the [Library
API](/docs/reference/api#run) for the exact surface.
