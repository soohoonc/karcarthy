---
title: Running Agents
description: Call an Agent and inspect its output, usage, events, and failures.
---

`run!` calls an Agent with one input value:

```clojure
(def run
  (k/run! researcher "How does ACP model selection work?"))
```

The usual input is a prompt string. An Agent may declare any `:input` contract,
including an object-shaped JSON Schema for structured input.

## The result

`run!` returns a Run rather than only the model's text:

```clojure
{:karcarthy/type :run
 :id "run_..."
 :status :completed
 :agent "researcher"
 :input "How does ACP model selection work?"
 :output "..."
 :usage {:model-calls 4
         :input-tokens 12000
         :output-tokens 900
         :generated-forms 0
         :duration-ms 4200.0}
 :events [...]
 :error nil}
```

`:status` is `:completed`, `:failed`, or `:cancelled`. A failed Run keeps its
usage, events, and structured `:error`; it does not throw ordinary model or Tool
failures past the `run!` boundary.

## The model and Tool loop

For an Agent with a model, `run!`:

1. validates the Agent input;
2. resolves its instructions, Tools, and available Agents;
3. calls the model with the prompt and callable schemas;
4. executes any Tool or Agent calls and returns their results to the model;
5. repeats until the model returns a final answer; and
6. validates that answer against `:output`.

Available Agents and generated Agents execute inside this Run, so their model
calls, tokens, events, deadlines, and cancellation contribute to the same
record. They do not inherit the parent's model conversation: each receives
only the input supplied in its Tool call.

## Compose Runs with Clojure

Application code can call Agents in any Clojure control flow:

```clojure
(defn completed-output [run]
  (if (= :completed (:status run))
    (:output run)
    (throw (ex-info "Agent failed" {:run run}))))

(defn review-change [change]
  (let [reviews (->> [security-reviewer api-reviewer]
                     (mapv #(future (k/run! % change)))
                     (mapv deref)
                     (mapv completed-output))]
    (k/run! editor {:change change :reviews reviews})))
```

Each explicit `run!` above creates an independent Run. This is appropriate
when Clojure chooses the control flow. Put Agents in a parent's `:agents`
vector instead when the parent model should decide whether and when to call
them inside one Run.

## Local context

Use `:context` for local dependencies that should not be sent automatically to
the model:

```clojure
(k/run! account-agent "Summarize account 123."
        {:context {:account-id "123"
                   :db db}})
```

Tools and Agent programs read it with `(k/context)`.

## Limits and cancellation

```clojure
(k/run! agent prompt
        {:limits {:model-calls 100
                  :input-tokens 1000000
                  :output-tokens 100000
                  :agent-depth 8
                  :parallelism 6
                  :generated-forms 20
                  :deadline-ms 300000}
         :cancel cancelled?})
```

`:cancel` may be a Boolean, predicate, atom, or another dereferenceable value.
Cancellation is checked around Agent, model, and Tool calls.

Other Run options are `:session`, `:observe`, `:approval`, `:metadata`,
`:model-transports`, and `:evaluation-namespace`. See the [Library
API](/docs/reference/api#run).
