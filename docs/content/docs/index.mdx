karcarthy is a small Clojure agent harness inspired by Lisp's homoiconicity.

Most agent libraries add a workflow graph or orchestration DSL beside the host
language. karcarthy uses Clojure itself. An Agent is a Clojure value; ordinary
functions, macros, conditionals, recursion, and concurrency are the workflow
engine.

```clojure
(require '[karcarthy :as k])

(k/defagent assistant
  {:model {:transport :responses :id "gpt-5.6"}
   :instructions "Answer clearly and concisely."
   :output string?})

(def run
  (k/run! assistant "Explain continuation-passing style."))

(:output run) ;=> "..."
```

`run!` sends the Agent's instructions and the prompt to the model. If the model
calls a Tool, karcarthy runs it and returns its result to the model. When the
model answers, `run!` returns a Run containing the final output, usage, events,
and any failure.

## Tools and available Agents

Tools are Clojure functions the model may call. Agents can also make other
Agents available by listing them in `:agents`:

```clojure
(k/defagent researcher
  {:description "Research one question and return sourced findings."
   :model {:transport :responses :id "gpt-5.6"}
   :instructions "Research the assigned question."
   :output string?})

(k/defagent editor
  {:model {:transport :responses :id "gpt-5.6"}
   :instructions "Write the final answer. Ask the researcher when needed."
   :agents [researcher]
   :output string?})
```

The editor's model sees `researcher` with its description and input schema. If
it calls `researcher`, that Agent runs, its result is returned to the
editor's model, and the editor continues. This conversion to a model Tool is an
implementation detail; users declare the relationship as `:agents
[researcher]`.

## Why homoiconicity matters

An Agent can also be a Clojure program that calls other Agents:

```clojure
(k/defagent review-team
  {:input string? :output vector?}
  [change]
  (->> [security-reviewer api-reviewer]
       (mapv #(future (k/run! % change)))
       (mapv deref)
       (mapv :output)))
```

That `agent` form is executable code and Clojure data. A developer can write
it, a macro can transform it, or a model can write a new form through the
built-in `agent` capability. karcarthy reads, expands, evaluates, verifies, and
runs the new Agent. Authored and generated orchestration therefore share one
language and one representation; there is no second workflow format.
