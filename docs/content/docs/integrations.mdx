---
title: Integrations
description: Connect model endpoints and MCP Tools, or expose an Agent over ACP.
---

karcarthy keeps integrations at three boundaries:

- model transports translate model requests and responses;
- external capabilities become Tools;
- ACP exposes an Agent to another process.

## Model endpoints

The built-in Responses transport defaults to OpenAI:

```clojure
{:transport :responses
 :provider :openai
 :id "gpt-5.6"}
```

It reads `RESPONSES_API_KEY` or `OPENAI_API_KEY`. Streaming is enabled by
default.

A Responses-compatible endpoint uses the same transport with a different URL
and credential source:

```clojure
{:transport :responses
 :provider :anthropic
 :id "anthropic/claude-model"
 :base-url "https://ai-gateway.vercel.sh/v1"
 :api-key-env "AI_GATEWAY_API_KEY"}
```

Endpoint options include `:base-url`, `:path`, `:url`, `:api-key`,
`:api-key-env`, `:headers`, `:auth?`, `:timeout-ms`, `:stream`, and
`:provider-options`.

Use `fake-model` for deterministic tests without a network call:

```clojure
(def model
  (k/fake-model
   (fn [_]
     {:type :final
      :output "deterministic response"})))
```

## Hosted Tools

Provider-hosted Tools are executed by the model endpoint:

```clojure
(k/defagent researcher
  {:model {:transport :responses :id "gpt-5.6"}
   :instructions "Research current information and cite sources."
   :tools [(k/responses-web-search)]
   :output string?})
```

The selected endpoint and model must support the capability.

## MCP

Connect to a stdio MCP server, discover its Tools, and close the connection when
finished:

```clojure
(def connection
  (k/connect-mcp!
   {:name "project"
    :command "npx"
    :args ["-y" "@example/project-mcp"]
    :cwd "/workspace/project"}))

(try
  (let [tools (k/mcp-tools connection)
        agent (k/agent
               {:name "project-agent"
                :model {:transport :responses :id "gpt-5.6"}
                :instructions "Inspect the project and answer the request."
                :tools tools
                :output string?})]
    (k/run! agent "Summarize the open work."))
  (finally
    (k/close-mcp! connection)))
```

MCP Tools use the same validation, approval, and event path as local Tools.
They require approval by default. The client implements MCP `2025-11-25` over
stdio.

## ACP

ACP exposes a karcarthy Agent to editors, evaluators, and other process-level
clients:

```clojure
(k/serve-acp! {:agent assistant})
```

Use a factory when each ACP session supplies a working directory, model, or MCP
Tools:

```clojure
(defn acp-agent [{:keys [cwd mcp-tools model-id]}]
  (k/agent
   {:name "coding-agent"
    :model {:transport :responses :id model-id}
    :instructions "Complete the coding request."
    :tools (into (k/local-tools {:cwd cwd}) mcp-tools)
    :output string?}))

(k/serve-acp! {:agent acp-agent
               :models ["gpt-5.6" "gpt-5.5"]})
```

Launch a namespace var as a stdio server:

```bash
clojure -M -m karcarthy.acp example/acp-agent
```

Run the server in its own process. The default server owns standard input and
output.

## Harbor

Harbor can treat the ACP server as an external Agent. karcarthy supplies Agent
messages, Tool updates, selected-model information, and reported token usage
over ACP. Harbor owns tasks, scorers, rewards, ATIF conversion, and experiment
storage.

There is no Harbor-specific trajectory or storage format in karcarthy. Packaging
an ACP launcher and selecting Harbor tasks are deployment and evaluation setup,
not additional harness abstractions.
