This example uses the implemented programming model. The concrete
provider and search functions are application dependencies.

## Contracts

```clojure
(ns example.research
  (:require [clojure.spec.alpha :as s]
            [karcarthy :as k]))

(s/def ::question string?)
(s/def ::request (s/keys :req-un [::question]))
(s/def ::answer string?)
(s/def ::sources (s/coll-of string? :kind vector?))
(s/def ::report (s/keys :req-un [::answer ::sources]))
(s/def ::accepted? boolean?)
(s/def ::feedback string?)
(s/def ::review (s/keys :req-un [::accepted?]
                        :opt-un [::feedback]))
```

## Tools

```clojure
(k/deftool web-search
  {:description "Search the web for evidence relevant to a question."
   :input (s/keys :req-un [::question])
   :output ::sources
   :approval :never}
  [rt {:keys [question]}]
  (search-web! (:http (k/context rt)) question))

(k/deftool fetch-page
  {:description "Fetch a page selected from search results."
   :input ::url-request
   :output string?}
  [rt {:keys [url]}]
  (fetch! (:http (k/context rt)) url))
```

## Leaf agents

```clojure
(k/defagent researcher
  {:description "Produce one evidence-backed report."
   :model {:transport :responses :id "gpt-5.6" :reasoning :high}
   :instructions "Research carefully. Separate evidence from inference."
   :input ::request
   :tools [web-search fetch-page]
   :output ::report})

(k/defagent fact-checker
  {:model {:transport :responses :id "gpt-5.6"}
   :instructions "Check every factual claim against the supplied sources."
   :input ::report
   :tools [fetch-page]
   :output ::review})

(k/defagent citation-checker
  {:model {:transport :responses :id "gpt-5.6"}
   :instructions "Check that each citation supports its associated claim."
   :input ::report
   :tools [fetch-page]
   :output ::review})

(k/defagent editor
  {:model {:transport :responses :id "gpt-5.6"}
   :instructions "Revise the report using the reviews without inventing evidence."
   :input ::edit-request
   :output ::report})
```

## The program

```clojure
(k/defagent research-team
  {:description "Research, independently verify, and revise a question."
   :input ::request
   :output ::report}
  [rt request]
  (let [draft (k/invoke! rt researcher request)
        reviews
        (k/await-all!
         [(k/spawn! rt fact-checker draft)
          (k/spawn! rt citation-checker draft)])]
    (if (every? :accepted? reviews)
      draft
      (k/invoke! rt editor {:draft draft :reviews reviews}))))
```

`research-team` has no model configuration because its body is the program.
Its children may be model-backed agents, custom programs, or generated agents.

## A generated specialist

An architect agent may return a Clojure source string:

```clojure
(agent
  {:name "domain-specialist"
   :model {:transport :responses :id "gpt-5.6"}
   :instructions "Analyze the domain terminology before answering."
   :tools [web-search fetch-page]
   :input ::request
   :output ::report})
```

The parent evaluates it like any other form:

```clojure
(let [source (k/invoke! rt architect request)
      child  (k/compile-agent! rt source)]
  (k/invoke! rt child request))
```

There is no generated-workflow protocol. The source is normal Clojure and may
contain a custom body, macros, nested agent forms, or structural transformations.

## Run and observe

```clojure
(def events (atom []))

(def result
  (k/run! research-team
          {:question "How do language models use tool feedback?"}
          {:context {:http http-client}
           :limits {:model-calls 40
                    :parallelism 2
                    :agent-depth 5
                    :deadline-ms 180000}
           :observe #(swap! events conj %)}))

(:status result) ;=> :completed
(:output result) ;=> {:answer "..." :sources [...]}
```

The event stream contains model calls, tool calls, child lineage, usage, and
any `:program/read`, `:program/expanded`, and `:program/evaluated` events from
generated agents. A research harness can compare these events without changing
the application program.
