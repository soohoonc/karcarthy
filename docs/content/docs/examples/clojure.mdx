---
title: Multi-Agent review
description: Build a coding review system with local Tools, parallel reviewers, and optional model-created Agents.
---

This example builds a review system for a local repository. The returned Agent
can be run directly or served over ACP.

## Build the system

```clojure
(ns example.review
  (:require [karcarthy :as k]))

(defn review-system [cwd]
  (let [tools (k/local-tools
               {:cwd cwd
                :approval {:read :never
                           :search :never
                           :write :always
                           :edit :always
                           :bash :always}})

        security-reviewer
        (k/agent
         {:name "security-reviewer"
          :model {:transport :responses :id "gpt-5.6"}
          :instructions
          "Inspect the change for concrete security problems. Cite files."
          :tools tools
          :input map?
          :output string?})

        api-reviewer
        (k/agent
         {:name "api-reviewer"
          :model {:transport :responses :id "gpt-5.6"}
          :instructions
          "Inspect the change for API compatibility problems. Cite files."
          :tools tools
          :input map?
          :output string?})

        editor
        (k/agent
         {:name "review-editor"
          :model {:transport :responses :id "gpt-5.6"}
          :instructions
          "Combine the reviews. Remove duplicates and unsupported claims."
          :input map?
          :output string?})]

    (k/agent
     {:name "review-team"
      :input map?
      :output string?}
     [rt change]
     (let [reviews
           (k/await-all!
            [(k/spawn! rt security-reviewer change)
             (k/spawn! rt api-reviewer change)])]
       (k/invoke! rt editor {:change change
                             :reviews reviews})))))
```

The reviewers use the standard model/Tool loop. `review-team` has a Clojure
body and coordinates them deterministically. All children share Run limits,
cancellation, usage accounting, and event storage.

## Run it

```clojure
(def team (review-system "/workspace/project"))

(def run
  (k/run! team
          {:request "Review the current working tree."
           :base "origin/main"}
          {:limits {:parallelism 2
                    :model-calls 20
                    :deadline-ms 180000}
           :approval
           (fn [{:keys [tool input]}]
             (confirm-tool-call? (:name tool) input))
           :observe handle-event}))

(:status run)
(:output run)
```

Only the local Tools configured with `:always` call the approval handler.

## Add an adaptive architect

For tasks where the useful specialist is not known in advance, create a
model-backed entry point with the Agent Tool:

```clojure
(defn adaptive-reviewer [cwd]
  (let [tools (k/local-tools {:cwd cwd})]
    (k/agent
     {:name "adaptive-reviewer"
      :model {:transport :responses :id "gpt-5.6"}
      :instructions
      (str "Review the repository. Call the agent Tool when a focused "
           "specialist or multi-Agent plan would improve the review.")
      :tools (conj tools (k/agent))
      :output string?})))

(k/run! (adaptive-reviewer "/workspace/project")
        "Review the current change."
        {:limits {:agent-depth 4
                  :generated-forms 8
                  :parallelism 3
                  :model-calls 40}})
```

The model can supply a new model-backed Agent or an Agent with a Clojure body.
Generated source can refer to application vars only when they are available in
the Run's evaluation namespace.

## Serve it over ACP

Use an Agent factory so ACP supplies the session working directory and MCP
Tools:

```clojure
(defn acp-reviewer [{:keys [cwd mcp-tools model-id]}]
  (let [local (k/local-tools {:cwd cwd})]
    (k/agent
     {:name "reviewer"
      :model {:transport :responses :id model-id}
      :instructions (k/system-prompt)
      :tools (into (conj local (k/agent)) mcp-tools)
      :output string?})))
```

```bash
clojure -M -m karcarthy.acp example.review/acp-reviewer
```
