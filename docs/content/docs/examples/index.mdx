The public examples use the live Responses transport. Fake models stay in the
offline unit suite, where they make kernel behavior deterministic and free.

| Example | What it demonstrates |
| --- | --- |
| [Basic Agent](/docs/quickstart#run-the-live-example) | One live model Run |
| [Chat](/docs/examples/chat) | Sessions and an interactive application |
| [Agent composition](/docs/examples/composition) | Predefined Agents coordinated with Clojure |
| [Coding Agent](/docs/examples/coding-agent) | Open-ended repository work with optional Agent generation |
| [Harbor](/docs/harbor#from-demonstration-to-search) | Search over executable Agent programs using rewards and trajectories |

The Coding Agent is the main end-to-end application example. The Harbor example
then treats a complete Clojure Agent form as the search artifact and uses
isolated tasks, verifier rewards, and trajectories to evolve it.

The [REPL guide](/docs/repl) covers interactive development. Integration details
for [MCP](/docs/mcp) and [ACP](/docs/acp) remain in their dedicated guides.
