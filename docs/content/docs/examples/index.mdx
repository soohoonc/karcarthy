Examples should show the power of the system first: agent workflows as data that
can plan, fan out, reduce, critique, and run again.

## Deep research

This is the serious example. It models the public shape of a deep-research
agent loop as EDN:

```text
question
  -> research-planner
  -> delegate(max 4)
       source-scout -> evidence-reader -> source-judge
  -> report-writer
  -> report-critic
       accept, or send feedback into another writer round
```

The workflow itself is just a value:

```clojure title="examples/clojure/deep_research.clj"
(def investigation-pipeline
  (k/pipe source-scout evidence-reader source-judge))

(def deep-research-workflow
  (k/revise
   (k/reduce
    (k/delegate research-planner investigation-pipeline :max-concurrency 4)
    report-writer)
   report-critic
   :max-rounds 2))
```

That small expression gives you:

| Node | Job |
| --- | --- |
| `research-planner` | Emits EDN subtasks, not prose. |
| `delegate` | Runs each planned track through the same investigation pipeline. |
| `source-scout` | Finds candidate sources for one track. |
| `evidence-reader` | Extracts claims and ties each claim to a URL. |
| `source-judge` | Drops unsupported evidence before synthesis. |
| `reduce` | Sends the accepted evidence packet to one report writer. |
| `report-critic` | Returns `{:accept? true}` or revision feedback. |
| `revise` | Re-runs the writer until the critic accepts or the round limit is hit. |

Run the deterministic offline version:

```bash
clojure -M -e '(load-file "examples/clojure/deep_research.clj")'
```

The file also includes a live Codex CLI runner path. karcarthy still owns only
the workflow value; Codex owns the inner model/tool loop:

```bash
KARCARTHY_CODEX_LIVE=1 clojure -M -e '(load-file "examples/clojure/deep_research.clj")'
```

The important part is that the graph can be printed, stored, diffed, inspected,
rewritten, validated, and observed before any live model runs.

```clojure
(k/agents deep-research-workflow)
;;=> [research-planner source-scout evidence-reader source-judge
;;    report-writer report-critic]

(k/workflow? deep-research-workflow)
;;=> true
```

## Launch readiness

This example shows dynamic routing plus parallel review:

```text
request
  -> classifier
       :launch
         -> branch(max 4)
              product-reviewer
              engineering-reviewer
              security-reviewer
              support-reviewer
         -> brief-writer
         -> critic
              accept, or revise once
       :incident
         -> incident-responder
```

The orchestration stays compact:

```clojure title="examples/clojure/launch.clj"
(def review-packet
  (k/branch reviewers :max-concurrency 4))

(def launch-brief
  (k/revise
   (k/pipe review-packet brief-writer)
   critic
   :max-rounds 2))

(def workflow
  (k/route classifier
           {:launch launch-brief
            :incident incident-responder}
           :default launch-brief))
```

Run it offline:

```bash
clojure -M -e '(load-file "examples/clojure/launch.clj")'
```

This is the pattern to copy when an app needs multiple specialized reviewers,
a synthesized decision, and a hard acceptance gate.

## What this proves

| Capability | Why it matters |
| --- | --- |
| Workflow as EDN/JSON data | The plan can be generated by code or by an agent and then checked before execution. |
| Small workflow vocabulary | `pipe`, `branch`, `delegate`, `reduce`, `revise`, `route`, `continue`, and `dynamic` cover sequencing, fan-out, synthesis, loops, routing, handoff, and adapting a run in progress. |
| Runners at the leaves | Claude CLI, Codex CLI, OpenAI Agents SDK, or a mock can run the same agent nodes. |
| Rewrites before execution | Add models, runner keys, instruction suffixes, or observability settings across a whole workflow. |
| Observable runs | `:observe` receives structured start/end/error events for agents and composite nodes. |

## Small checks

Use these only when you want to verify installation:

```bash
./bin/karcarthy agent echo --instructions "Echo the input." hi
./bin/karcarthy run examples/workflows/echo.json hi
clojure -M -m karcarthy.demo
```

Expected output for the first two commands:

```text
[echo] hi
```
