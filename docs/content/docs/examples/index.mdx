## Launch Readiness

This is the main product-style example. Given a product-change prompt, it:

1. classifies the request as launch or incident,
2. runs product, engineering, security, and support reviewers,
3. synthesizes their notes into a launch brief,
4. asks a critic to accept or request revisions.

The workflow shape is small:

```clojure
(def reviewers
  [product-reviewer engineering-reviewer security-reviewer support-reviewer])

(def review-packet
  (k/branch reviewers :max-concurrency 4))

(def launch-brief
  (k/revise
   (k/pipe review-packet brief-writer)
   critic
   :max-rounds 2))

(def workflow
  (k/route classifier
           {:launch launch-brief
            :incident incident-responder}
           :default launch-brief))
```

Then the caller chooses a runner. The offline run uses `mock-runner`, which
takes a respond fn of `{:agent :prompt :options} -> String` — not a map of
canned responses:

```clojure
;; respond: fn of {:agent :prompt :options} -> reply string.
;; launch.clj dispatches on (:name agent) to return canned reviewer notes.
(k/run {:runner (k/mock-runner respond)
        :workflow workflow
        :input "Prepare the launch brief for a new enterprise SSO feature."})
```

Run the same example in each host language:

```bash
clojure -M -e '(load-file "examples/clojure/launch.clj")'
python3 examples/python/launch.py
npx tsx examples/typescript/launch.ts
```

## Deep Research

This example models the public shape of a deep-research agent loop as EDN:

```text
question
  -> research-planner
  -> delegate(max 4)
       source-scout -> evidence-reader -> source-judge
  -> report-writer
  -> report-critic
       accept, or send feedback into another writer round
```

The workflow itself is just a value:

```clojure title="examples/clojure/deep_research.clj"
(def investigation-pipeline
  (k/pipe source-scout evidence-reader source-judge))

(def deep-research-workflow
  (k/revise
   (k/reduce
    (k/delegate research-planner investigation-pipeline :max-concurrency 4)
    report-writer)
   report-critic
   :max-rounds 2))
```

Run the deterministic offline version:

```bash
clojure -M -e '(load-file "examples/clojure/deep_research.clj")'
```

The file also includes a live `codex-runner` path. karcarthy still owns only
the workflow value; Codex owns the inner model/tool loop:

```bash
KARCARTHY_CODEX_LIVE=1 clojure -M -e '(load-file "examples/clojure/deep_research.clj")'
```

## More Examples

The repository ships more complete examples than the two walkthroughs above.
Unless marked live, each runs offline with no API keys.

- `examples/clojure/claude_dynamic_agents.clj` — Claude dynamic-workflow style
  orchestration: a lead agent sharpens its own instructions with the
  experimental `evolve`, a
  planner creates workstreams at runtime (`delegate` into a worker `pipe`,
  synthesized with `reduce`), and a critic accepts or requests one revision
  (`revise`). Offline by default; set `KARCARTHY_CLAUDE_LIVE=1` to run through
  the Claude CLI.
- `examples/javascript/claude_dynamic_agents.mjs` — the `delegate`/`reduce`/
  `revise` core of the same workflow built as JSON and sent through
  `bin/karcarthy` (the `evolve` lead stage is Clojure-only); `--print` shows
  the workflow JSON without invoking Clojure.
- `examples/clojure/rewrite.clj` — build a workflow once, then `configure` a
  model and an instruction suffix onto every agent before interpreting the
  rewritten value. The runnable version of the rewrites row below.
- `examples/clojure/openai_deep_research.clj` — wraps OpenAI Deep Research
  (the Responses API) as a karcarthy leaf. Offline by default; set
  `KARCARTHY_OPENAI_LIVE=1` with `OPENAI_API_KEY` for a live background
  research job.
- `examples/javascript/openai_deep_research.mjs` — the JavaScript variant;
  `--request` prints the live request JSON without calling the API.
- `examples/clojure/evolve.clj` — a live `evolve` run (paid `claude -p`): an
  agent patches its own instructions via EDN at runtime, then answers with the
  new behavior.
- `examples/clojure/live.clj` — a live `delegate`/`reduce` run through
  `claude-runner` (paid `claude -p`), showing the CLI tuning
  (`:system-prompt-mode :replace`, tools disabled) that makes Claude answer
  directly instead of inheriting Claude Code's interactive persona.
- `examples/java/Demo.java`, `examples/kotlin/Demo.kt`, and
  `examples/scala/Demo.scala` — JVM host demos: build two agents, compose a
  `pipe`, and run it on the mock runner through the Clojure Java API.

## What These Prove

| Capability | Why it matters |
| --- | --- |
| Workflow as EDN/JSON data | The plan can be generated by code or by an agent and checked before execution. |
| Small workflow vocabulary | `pipe`, `branch`, `delegate`, `reduce`, `revise`, `route`, `continue`, and `dynamic` cover sequencing, fan-out, synthesis, loops, routing, handoff, and adapting a run in progress. |
| Runners at the leaves | Claude, Codex, OpenAI, ACP, a process command, or a mock can run the same agent nodes. |
| Rewrites before execution | Add models or instruction suffixes across a whole workflow with `configure`, or make arbitrary structural edits with `over`. |
| Observable runs | `:observe` receives structured start/end/error events for agents and composite nodes. |
