---
title: TypeScript over ACP
description: Use a karcarthy agent from a TypeScript application through its ACP server.
---

TypeScript applications interact with a running karcarthy harness through ACP.
The agent program remains Clojure code; TypeScript supplies tasks and handles
events, approvals, cancellation, and results.

## Start a session

The concrete method names depend on the ACP client package. Conceptually:

```typescript
await using process = await acp.spawn({
  command: "clojure",
  args: [
    "-M",
    "-m",
    "karcarthy.acp",
    "example.research/research-team",
  ],
});

const session = await process.newSession({
  cwd: "/workspace",
});

for await (const event of session.prompt([
  { type: "text", text: "How do language models use tool feedback?" },
])) {
  render(event);
}
```

The application can render tool progress and approvals without
understanding or serializing the Clojure program.

## Stable interface, evolvable program

The server may load a handwritten agent, a macro-generated agent, or an agent
that constructs more agents at runtime. All of them present the same ACP
session interface:

```text
TypeScript client
        │ ACP
        ▼
Clojure harness ── run! ── Agent
                         ├─ model calls
                         ├─ tools
                         ├─ child agents
                         └─ generated Clojure forms
```

No JSON workflow schema sits between the client and the harness.

## Cancellation, approvals, and MCP

The server translates `session/cancel` into the root Runtime's cancellation
signal, sends approval decisions through `session/request_permission`, and
maps Tool events to ACP updates. Stdio MCP servers supplied in `session/new`
are initialized and their tools are added to the session Agent. Durable
approval resumption and HTTP/SSE MCP are separate roadmap work.
