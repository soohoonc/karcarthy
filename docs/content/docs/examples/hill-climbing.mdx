---
title: Hill Climbing
description: Compare generated Agent programs locally, then repeat the search with Harbor.
---

The Dynamic Agents example proves that karcarthy can compile and run a
model-authored Agent. This example asks the next research question: can an
outer search retain a better Agent program using an outcome metric?

Three deterministic candidates receive the same three maintenance tasks:

| Candidate | Generated behavior | Expected reward |
| --- | --- | ---: |
| `noop` | Leaves the workspace unchanged. | 0 / 3 |
| `literal` | Repairs only retry backoff. | 1 / 3 |
| `patcher` | Repairs all three implementations. | 3 / 3 |

Each task contains a broken `main.py`: `retry` covers capped exponential
backoff, `redact` covers recursive secret masking, and `ledger` covers settled
transaction reconciliation. Harbor grades behavior with Python assertions
rather than comparing a hidden answer string. Each task also carries an oracle
solution for validating the environment and verifier independently.

The candidates are intentionally simple. The point is to expose the complete
generate → run → measure → select loop without attributing random model
variation to the harness.

## Run locally

The fast path requires only JDK 21 and the Clojure CLI:

```bash
clojure -M:examples hill-climb
```

Observed output:

```text
metric: exact reference patch match
noop           [...] 0/3
literal        [#..] 1/3
patcher        [###] 3/3
winner: patcher
```

Every candidate is submitted through the built-in `agent` Tool and run by the
ordinary karcarthy kernel. The fast local evaluator compares the resulting
`main.py` with a reference patch and writes trial records under
`target/hill-climb-results/`.

Use this mode while developing the Agent forms or metric. It is deterministic,
offline, and quick enough for the normal test suite.

## Run with Harbor

The Harbor mode evaluates the same candidates and tasks in isolated
environments:

```bash
examples/harbor/hillclimb.sh
```

It replaces the local source comparison with behavioral unit tests and adds
verifier-owned rewards, job metadata, raw ACP events, and ATIF trajectories.
The verified Harbor 0.18.0 run completed all nine trials without exceptions and
selected the same `patcher` candidate.

Use Harbor when isolation, portable trajectories, or a results UI matters.
Inspect the completed jobs with Harbor's built-in viewer:

```bash
uvx --python 3.13 --from harbor \
  harbor view examples/harbor/jobs
```

The [Harbor integration guide](/docs/harbor) documents the ACP packaging
boundary, recorded artifacts, registry status, and experimental metadata in
detail. The runnable files live under `examples/harbor`.

## What this establishes

This example demonstrates susceptibility to metric-driven search: candidate
Agent programs produce different rewards, and an outer loop can retain the
highest-scoring form. It does not establish that these three hand-authored
candidates constitute a novel optimizer, nor that the result generalizes to
stochastic model-generated populations. Harbor makes those larger experiments
reproducible without changing karcarthy's execution model.
