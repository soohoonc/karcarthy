---
title: Python over ACP
description: Drive and evaluate a running karcarthy harness from Python without turning the program into JSON.
---

Python is an external client of the Clojure harness. It does not reconstruct an
agent program from JSON or select a runner. The Clojure process loads and
executes the program, while Agent Client Protocol (ACP) carries session input,
events, approvals, cancellation, and results.

## Start the harness

The command is:

```bash
clojure -M -m karcarthy.acp example.research/research-team
```

The process speaks ACP over stdio. The selected var must contain an Agent or a
function that builds an Agent from the ACP session context.

## Connect from Python

The exact names depend on the ACP client library. Conceptually:

```python
async with acp.spawn(
    ["clojure", "-M", "-m", "karcarthy.acp",
     "example.research/research-team"]
) as agent:
    session = await agent.new_session(
        cwd="/workspace",
    )

    async for event in session.prompt([
        {"type": "text",
         "text": "How do language models use tool feedback?"}
    ]):
        record(event)
```

The client sees tool-call updates, permission requests, streaming message
chunks, and the prompt stop reason. The server retains the Clojure source form,
expanded form, local context, model credentials, Tools, and one in-memory
conversation Session per ACP session.

## Harbor evaluation

Harbor can launch the same ACP server for every task. An experiment varies the
loaded agent var, model configuration, generated-code policy, or source
transformation while keeping the protocol stable.

Useful measurements include:

- task reward or verifier outcome;
- tool and model call counts;
- token cost and latency;
- generated program count and depth;
- approval, cancellation, and failure behavior;
- differences between source and expanded forms;
- behavioral invariants across program variants.

The evaluation boundary does not require a portability abstraction inside the
harness.

## What crosses ACP

| Crosses the boundary | Stays in Clojure |
| --- | --- |
| prompts and structured task input | executable agent bodies |
| ACP tool and message updates | function values and closures |
| approval and input responses | tools and local dependencies |
| cancellation | macroexpansion and evaluation |
| final Agent message and stop reason | Session history, usage, and provider credentials |
