---
title: Quickstart
description: Run one agent, compose a tiny workflow, and choose a runner at call time.
---

The whole API starts here:

```clojure
(k/run runner workflow input)
```

`workflow` can be a single agent. `runner` is the thing that actually runs that
agent. The workflow does not choose its own backend.

## Requirements

- JDK 21 or newer
- Clojure CLI
- A checkout of `soohoonc/karcarthy`

## Run One Agent

```clojure
(require '[karcarthy :as k])

(k/defagent echo
  {:instructions "Echo the input."})

(k/run (k/mock-runner) echo "hi")
```

Result:

```clojure
{:karcarthy/type :result
 :ok? true
 :agent "echo"
 :text "[echo] hi"
 :raw {:runner :mock}}
```

That is the smallest workflow: one agent node, one runner, one input.

## Compose Two Agents

```clojure
(k/defagent researcher
  {:instructions "Research the question and list the important facts."})

(k/defagent summarizer
  {:instructions "Summarize the previous answer in one sentence."})

(def workflow
  (k/pipe researcher summarizer))

(k/run (k/mock-runner)
       workflow
       "why does Lisp use code as data?")
```

`pipe` feeds the first result's `:text` into the second agent.

## Choose The Runner At Runtime

Keep agents runner-agnostic:

```clojure
(k/defagent reviewer
  {:instructions "Review the input and return concrete issues."
   :model "sonnet"})
```

Pick the runner before calling `run`:

```clojure
(defn review [runner input]
  (k/run runner reviewer input))

(review (k/mock-runner) "check this design")
(review (k/claude-cli-runner {}) "check this design")
(review (k/acp-runner {:command ["my-acp-agent"]}) "check this design")
```

Agents do not carry a backend selector. If the runner changes at runtime, pass
a different runner value.

## CLI

Build the CLI jar:

```bash
clojure -T:build uber
```

Run one mock agent:

```bash
./bin/karcarthy agent echo --instructions "Echo the input." hi
```

Expected output:

```text
[echo] hi
```

Choose the CLI runner for the whole invocation with `--runner`:

```bash
./bin/karcarthy agent reviewer \
  --runner claude \
  --instructions "Review the input and return concrete issues." \
  "check this design"
```
