---
title: Workflow Data
description: EDN shapes for agents, workflows, and results.
---

Workflows are EDN values. Every node is a map tagged with `:karcarthy/type`.
The code-level schema references are plain data:

```clojure
(require '[karcarthy :as k])

k/edn-schema
k/json-schema
```

Validate EDN workflows with `workflow?`.

## Agent

```clojure
{:karcarthy/type :agent
 :name "researcher"
 :description "Use for research tasks."
 :instructions "Research the question and cite sources."
 :model "sonnet"
 :tools ["mcp__feedback__search"]
 :config {:temperature 0.2}}
```

Required keys: `:name`, `:instructions`. `:model`, `:tools`, and `:config` are
interpreted by the runner passed to `run`.

## Subagent

Subagents are runner configuration, not workflow nodes:

```clojure
{:karcarthy/type :subagent
 :name "security-reviewer"
 :description "Use for auth, secrets, injection, permission, and data exposure review."
 :instructions "Review like a security owner. Return concrete findings with file refs."
 :model "sonnet"
 :tools ["Read" "Grep" "Glob"]
 :disallowed-tools ["Write" "Edit"]
 :mcp-servers ["github"]
 :permission-mode :plan
 :max-turns 4
 :skills ["review"]
 :effort :high
 :background? true
 :isolation :worktree}
```

Required keys: `:name`, `:description`, `:instructions`.

## pipe

Runs each step in order. The `:text` from one result becomes the input for the
next step.

```clojure
{:karcarthy/type :pipe
 :steps [WORKFLOW ...]}
```

## step

Runs a host Clojure function inside a workflow:

```clojure
{:karcarthy/type :step
 :f clojure.string/trim
 :name "trim"
 :context? false}
```

`step` is Clojure-only because `:f` is a function value. It is not represented
in the JSON schema.

## branch

Runs a collection of branches on the same input:

```clojure
{:karcarthy/type :branch
 :branches [WORKFLOW ...]
 :max-concurrency 16}
```

## delegate

Runs a worker once per subtask from a planner:

```clojure
{:karcarthy/type :delegate
 :planner PLANNER
 :worker WORKFLOW
 :max-concurrency 16}
```

`PLANNER` is a workflow whose result text is EDN:

```clojure
{:subtasks ["first" "second"]}
```

## reduce

Runs branch/delegate work, then passes an EDN summary to a reducer workflow.

```clojure
{:karcarthy/type :reduce
 :source {:karcarthy/type :branch ...}
 :reducer WORKFLOW}
```

The reducer receives EDN shaped like:

```clojure
{:input "original input"
 :subtasks ["first" "second"]
 :results [{:agent "worker" :ok? true :text "..."}]}
```

## revise

Runs a worker and evaluator until accepted or the round limit is reached.

```clojure
{:karcarthy/type :revise
 :worker WORKFLOW
 :evaluator EVALUATOR
 :max-rounds 3}
```

`EVALUATOR` is a workflow whose result text is an EDN verdict:

```clojure
{:accept? true}
{:accept? false :feedback "specific revision feedback"}
```

## route

Routes by the source result:

```clojure
{:karcarthy/type :route
 :source WORKFLOW
 :routes {:label WORKFLOW}
 :default WORKFLOW}
```

## continue

Continues from one workflow to another:

```clojure
{:karcarthy/type :continue
 :source WORKFLOW
 :to WORKFLOW
 :prompt "optional override"}
```

When `:routes` is present, `:source` replies with EDN `{:route :label}` and the
selected workflow receives the original input. Route matching is exact. When
`:to` is present, the second workflow receives the source result text unless
`:prompt` is provided.

## dynamic

Lets an agent define, patch, call, spawn, and complete work while a workflow run
is in progress:

```clojure
{:karcarthy/type :dynamic
 :agent AGENT
 :max-steps 25}
```

The agent emits EDN ops:

```clojure
{:op :define
 :agent {:name "reader"
         :instructions "..."
         :model "sonnet"
         :tools ["Read"]
         :config {}}}
{:op :define :name "review" :workflow WORKFLOW}
{:op :patch :agent "reader" :merge {:instructions "..."}}
{:op :call :workflow "review" :input "..."}
{:op :spawn :agent "reader" :inputs ["a" "b"]}
{:op :complete :text "final answer"}
```

Refs are only resolved inside a dynamic run:

```clojure
{:karcarthy/type :agent-ref :name "reader"}
{:karcarthy/type :workflow-ref :name "review"}
```

## Result

```clojure
{:karcarthy/type :result
 :ok? true
 :agent "name"
 :text "final reply"
 :raw {...}}
```

Composite nodes add more fields:

| Node | Extra fields |
| --- | --- |
| `pipe` | last child result fields. |
| `branch` | `:results`, `:text`. |
| `delegate` | `:subtasks`, `:results`, `:text`. |
| `reduce` | `:source`, `:reduced`, reducer `:text`. |
| `revise` | `:rounds`, `:accepted?`. |
| `route` | selected child result fields, or `:error :no-route`. |
| `continue` | continued workflow result fields. |
