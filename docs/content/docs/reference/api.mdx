The public API is intentionally small.

## Agents

### `agent`

```clojure
(k/agent)
(k/agent config)
(k/agent config [rt input] body ...)
```

With no arguments, returns the model-facing `agent` Tool: the recursive
homoiconic capability. Its input is `{:source <one (agent ...) form> :input
<value>}`; it reads, expands, checks, evaluates, and invokes the resulting
Agent in the current run tree.

With a config, constructs an Agent and retains the original and macroexpanded
forms. With no body, the Agent uses the harness's standard model/tool loop.
With a body, the body is the Agent's program and must return a value satisfying
`:output`.

Required configuration:

- `:name` when using `agent` directly;
- `:model` and `:context` when using the default loop.

Optional configuration includes `:description`, `:environment`, `:input`,
`:tools`, `:output`, `:loop`, `:guardrails`, `:limits`, and `:hooks`.

### `defagent`

```clojure
(k/defagent name config)
(k/defagent name config [rt input] body ...)
```

Defines a var containing an Agent. The symbol supplies the default `:name`.

### `agent?`

```clojure
(k/agent? value)
```

Returns true when the value satisfies the Agent contract.

### `source-form` and `expanded-form`

```clojure
(k/source-form agent)
(k/expanded-form agent)
```

Return the form captured before and after macroexpansion.

## Tools

### `tool`

```clojure
(k/tool config [rt input] body ...)
```

Constructs a Tool. The config requires `:name`, `:description`, and `:input`.
`:output`, `:approval`, `:enabled?`, `:guardrails`, `:timeout-ms`, `:retry`, and
`:to-model-output` are optional.

### `deftool`

```clojure
(k/deftool name config [rt input] body ...)
```

Defines a var containing a Tool. The symbol supplies the default model-facing
name.

### `tool?`

```clojure
(k/tool? value)
```

Returns true when the value satisfies the Tool contract.

### `hosted-tool` and `responses-web-search`

```clojure
(k/hosted-tool :responses {:type "web_search"})
(k/responses-web-search)
(k/responses-web-search {:search_context_size "low"})
```

A hosted Tool is executed by its matching transport rather than by karcarthy.
The selected transport rejects hosted specifications it does not own.
`hosted-tool?` identifies this value type.

### `as-tool`

```clojure
(k/as-tool agent)
(k/as-tool agent options)
```

Projects an Agent as a Tool. Options may override its tool name, description,
input contract, output contract, approval policy, and environment projection. The
parent remains active after the child returns.

## Execution

### `run!`

```clojure
(k/run! agent input)
(k/run! agent input options)
```

Creates a root Runtime, validates input and environment, executes the Agent,
and returns a Run record containing the latest conversation state.

Options:

```clojure
{:environment {...}
 :state previous-conversation-state
 :limits {...}
 :observe event-fn
 :approval approval-handler
 :cancel cancellation-token
 :metadata {...}}
```

### `invoke!`

```clojure
(k/invoke! rt agent input)
(k/invoke! rt agent input options)
```

Runs a child Agent inside an existing Runtime tree and returns its typed output.
The child remains visible through its lineage and lifecycle events in the root
Run trace.

### `spawn!`

```clojure
(k/spawn! rt agent input)
(k/spawn! rt agent input options)
```

Starts a child invocation under structured concurrency and returns a child
handle. The child inherits cancellation and receives a budget grant from the
parent.

### `await!` and `await-all!`

```clojure
(k/await! child)
(k/await-all! children)
```

Wait for one or several child handles. Failure and cancellation propagate as
structured exceptions unless the program catches them.

### `handoff!`

```clojure
(k/handoff! rt agent input)
```

Records an `:agent/handoff` event and invokes another Agent while retaining
runtime, budget, and trace lineage. Durable conversation transfer remains
roadmap work.

### `environment`

```clojure
(k/environment rt)
```

Returns the local dependency-injection value. It is not automatically exposed
to the model.

## Context and prompts

Agent `:context` is a string or a function called for every model turn. The
function receives `:runtime`, `:environment`, `:agent`, complete `:history`,
current `:pending` messages, provider `:state`, and `:turn`. It returns either a
string or the provider-neutral map `{:system string-or-nil :messages [...]}`.
Unknown keys fail closed.

### `prompt`, `prompt-file`, and `system-prompt`

```clojure
(def context
  (k/prompt
   (k/system-prompt)
   (k/prompt-file "AGENTS.md")
   (fn [{:keys [environment]}]
     (str "Current user: " (:user-id environment)))))
```

`prompt` joins strings and turn-dependent string functions. `prompt-file`
reads exactly the named UTF-8 file. `system-prompt` reads the packaged
`resources/karcarthy/system.md`. None of them infer a directory, project, tool
set, date, or hidden context source.

## Conversation state

### `conversation-state?`

```clojure
(def first-run (k/run! agent "first"))
(k/conversation-state? (:state first-run)) ;=> true
(def next-run (k/run! agent "next" {:state (:state first-run)}))
```

The snapshot is ordinary versioned data containing messages, model identity,
and provider continuation state. The harness does not choose a persistence
backend. It resumes the root model-backed Agent only; custom orchestration must
model and persist its own domain state explicitly.

## Generated code

### `read-agent-form`

```clojure
(k/read-agent-form source)
```

Reads one Clojure form from a source string with reader evaluation disabled.
Returns the form as data.

### `check-agent-form!`

```clojure
(k/check-agent-form! rt form)
```

Fully macroexpands executable positions in the evaluation namespace while
preserving quoted source as data, then returns the checked
representation or throws a structured expansion exception. Compiler symbol and
arity failures are reported by `eval-agent-form!`.

### `eval-agent-form!`

```clojure
(k/eval-agent-form! rt checked-form)
```

Evaluates a checked form in the Runtime's evaluation namespace and verifies
that the result is an Agent.

### `compile-agent!`

```clojure
(k/compile-agent! rt source)
```

Convenience composition of `read-agent-form`, `check-agent-form!`, and
`eval-agent-form!`. Emits source, expanded-form, analysis, evaluation, and
failure events.

## Model access

### `model!`

```clojure
(k/model! rt request)
```

Low-level access to the active model transport for custom Agent programs. Most
agents should use the standard loop or invoke a model-backed child instead.

Provider transports implement only the normalized model request boundary. They
never execute tools or child agents.

### `model-transport`

```clojure
(k/model-transport complete-fn)
(k/model-transport {:complete complete-fn
                    :stream (fn [request emit-delta!] ...)})
```

A streaming implementation emits normalized `:text-delta` or
`:tool-call-delta` values and returns the same final response required from
`complete-fn`. Set model option `:stream false` to force the complete path when
both are available.

### Responses-compatible transport

`:transport :responses` selects the built-in HTTP adapter. Direct OpenAI needs
only a model ID and `OPENAI_API_KEY`:

```clojure
{:transport :responses
 :provider :openai
 :id "gpt-5.6"}
```

A compatible gateway changes endpoint and credentials, not harness semantics:

```clojure
{:transport :responses
 :provider :anthropic
 :id "anthropic/claude-model"
 :base-url "https://ai-gateway.vercel.sh/v1"
 :api-key-env "AI_GATEWAY_API_KEY"
 :headers {"X-Application" "karcarthy"}}
```

Transport options:

- `:base-url` — API root; defaults to `https://api.openai.com/v1`;
- `:path` — endpoint path; defaults to `/responses`;
- `:url` — complete URL override;
- `:api-key` or `:api-key-env` — explicit credentials;
- `RESPONSES_API_KEY`, then `OPENAI_API_KEY` — environment fallbacks;
- `:headers` — additional or replacement HTTP headers;
- `:auth? false` — omit automatic Bearer authentication for a trusted local
  endpoint;
- `:timeout-ms` — complete request timeout;
- `:stream false` — use the non-streaming HTTP response path;
- `:provider-options` — additional Responses request-body fields.

Model IDs pass through unchanged. Endpoint-specific hosted Tools work only
when the selected endpoint and model implement their specifications. `:provider`
is optional descriptive metadata; `:transport` selects executable model I/O.

## Local capabilities

### `local-tools`

```clojure
(def tools (k/local-tools {:cwd "/workspace"}))

(k/agent {:name "local-agent"
          :model {:transport :responses :id "gpt-5.6"}
          :context (k/system-prompt)
          :tools tools
          :output string?})
```

`local-tools` returns `read`, `write`, `edit`, `bash`, and `search` rooted
at an existing directory. Its options configure approval and output/time
bounds. Tool schemas travel through the model request independently from
context. `local-tools` constructs ordinary Tools, not a special Agent type.

## MCP

### `connect-mcp!`, `mcp-tools`, and `close-mcp!`

```clojure
(def connection
  (k/connect-mcp! {:name "server"
                   :command "server-command"
                   :args ["--stdio"]
                   :cwd "/workspace"}))
(def tools (k/mcp-tools connection))
(k/close-mcp! connection)
```

The connection implements MCP `2025-11-25` over stdio. `mcp-tools` accepts
`:prefix` and `:approval`; approval defaults to `:always`. Always close the
connection. Commands are executed directly rather than through a shell. ACP
session MCP processes start in the session `cwd`.

## ACP

### `serve-acp!`

```clojure
(k/serve-acp! {:agent agent-or-factory})
```

Serves ACP v1 JSON-RPC over `System/in` and `System/out`, or `:in` / `:out`
streams supplied in the options. A factory receives `{:cwd ... :mcp-tools ...
:session-id ...}` and returns an Agent. `:run-options` supplies base Run
options; `:client-request-timeout-ms` bounds permission requests.

The executable entry point resolves a var and serves its Agent value or
factory:

```bash
clojure -M -m karcarthy.acp your.namespace/agent-var
```

## Observation

### `emit!`

```clojure
(k/emit! rt event)
```

Adds runtime identity and lineage to an event, appends it to the Run trace, and
passes it to the configured observer.

### `events`

```clojure
(k/events run)
```

Returns the recorded observation events from a Run or Runtime.

## Contracts

Agent, tool, environment, input, and output contracts initially use
`clojure.spec`. A contract adapter may support other Clojure schema systems
later, but the harness exposes one validation result shape.

See [Contracts](/docs/reference/schema) for exact values and failure data.
