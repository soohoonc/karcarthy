description: Signatures, parameters, return values, and examples for the public facade.
---

Use the facade namespace for normal application code:

```clojure
(require '[karcarthy :as k])
```

## Agents

### `agent`

```clojure
(k/agent spec)
```

Builds an agent spec map. The map form is the primary API.

| Key | Type | Required | Meaning |
| --- | --- | --- | --- |
| `:name` | string | yes | Stable agent name used in results, logs, and mock responses. |
| `:instructions` | string | yes | Developer instructions for this agent spec. |
| `:description` | string | no | Short human-facing description. |
| `:model` | string | no | Runner-specific model selector. |
| `:tools` | vector of strings | no | Tool allowlist. Tools must already exist in the selected runner environment. |
| `:runner` | keyword | no | Runner registry key used by `run`. |
| `:config` | map | no | Runner-specific configuration. |

Returns:

```clojure
{:karcarthy/type :agent
 :name "reviewer"
 :description "Use for launch-risk review."
 :instructions "Find launch risks."
 :model "sonnet"
 :tools ["mcp__feedback__search"]
 :runner :claude
 :config {:temperature 0.2}}
```

Example:

```clojure
(k/agent
 {:name "reviewer"
  :instructions "Find launch risks and propose mitigations."
  :model "sonnet"
  :tools ["mcp__feedback__search"]})
```

To derive a variant from existing agent data:

```clojure
(k/agent {:from reviewer
          :runner :openai
          :model "gpt-5.2"})
```

The old positional form `(k/agent name instructions & opts)` is deprecated.

### `defagent`

```clojure
(k/defagent symbol spec)
```

Defines a var containing an agent. The symbol name becomes the agent `:name`.

```clojure
(k/defagent researcher
  {:instructions "Research the question and cite sources."
   :model "sonnet"})

(k/defagent claude-researcher
  {:from researcher
   :runner :claude})
```

The old positional form `(k/defagent symbol instructions & opts)` is
deprecated.

### `agent?` and `explain-agent`

```clojure
(k/agent? value)
(k/explain-agent value)
```

`agent?` returns true for well-formed agent maps. `explain-agent` returns a
human-readable validation message or nil.

## Subagents

### `subagent`

```clojure
(k/subagent name description instructions & opts)
```

Builds runner-native delegation config. A subagent is not a workflow node; it
is made available inside one runner call when the runner supports native
subagents or handoffs.

```clojure
(def security-reviewer
  (k/subagent "security-reviewer"
    "Use for auth, secrets, injection, permission, and data exposure review."
    "Review like a security owner. Return concrete findings with file refs."
    :tools ["Read" "Grep" "Glob"]
    :model "sonnet"
    :effort :high))
```

Common options:

| Option | Meaning |
| --- | --- |
| `:model` | Runner-native model selector. |
| `:tools` | Tool allowlist. |
| `:disallowed-tools` | Tool denylist where supported. |
| `:mcp-servers` | MCP servers available to the subagent. |
| `:permission-mode` / `:sandbox-mode` | Provider permission or sandbox controls. |
| `:max-turns` | Turn cap for providers that support it. |
| `:skills` | Skills preloaded or enabled by providers that support skills. |
| `:effort` / `:reasoning-effort` | Provider reasoning effort controls. |
| `:background?` / `:isolation` | Provider concurrency or isolation controls. |

Runner lowering:

| Runner | Native primitive |
| --- | --- |
| `claude-cli-runner` | Claude Code `--agents` JSON. |
| `openai-agents-runner` | OpenAI Agents SDK `handoffs`. |
| `karcarthy.runner.codex/subagent->config` | Codex custom agent config keys for `.codex/agents/` or `~/.codex/agents/`. |
| `process-runner` | Unsupported; process commands stay fixed runner config. |

### `defsubagent`

```clojure
(k/defsubagent symbol description instructions & opts)
```

Defines a var containing a subagent. The symbol name becomes the subagent
`:name`.

### `subagent?` and `explain-subagent`

```clojure
(k/subagent? value)
(k/explain-subagent value)
```

## Workflows

### `pipe`

```clojure
(k/pipe & steps)
```

Runs steps in sequence. Each step receives the previous result's `:text`.

```clojure
(k/pipe researcher summarizer)
```

### `step`

```clojure
(k/step f & {:keys [name context?]})
```

Runs host Clojure code inside a workflow. By default, `f` receives the flowing
input string. With `:context? true`, `f` receives
`{:input input :opts opts :node node}`.

```clojure
(k/pipe
 writer
 (k/step clojure.string/trim :name "trim")
 reviewer)
```

`step` may return text, a partial result map, or a full result map. It is
Clojure-local and not EDN/JSON portable.

### `branch`

```clojure
(k/branch workflows & {:keys [max-concurrency]})
```

Runs every workflow in the collection on the same input.

```clojure
(k/branch [product-reviewer engineering-reviewer security-reviewer]
          :max-concurrency 4)
```

### `delegate`

```clojure
(k/delegate planner worker & {:keys [max-concurrency]})
```

Reads planner EDN `{:subtasks [...]}` and runs the worker once per subtask.

```clojure
(k/delegate planner worker :max-concurrency 8)
```

### `reduce`

```clojure
(k/reduce source reducer & {:keys [max-concurrency]})
```

Runs branch/delegate work, then sends an EDN result summary to a reducer workflow.

```clojure
(k/reduce
  (k/branch [product-reviewer engineering-reviewer])
  brief-writer)
```

The reducer receives EDN shaped like `{:input ... :subtasks ... :results [...]}`.

### `revise`

```clojure
(k/revise worker evaluator & {:keys [max-rounds]})
```

Runs a worker/evaluator loop until accepted or the round limit is reached.

```clojure
(k/revise draft-writer critic :max-rounds 3)
```

Evaluator workflows reply with EDN verdict maps:

```clojure
{:accept? true}
{:accept? false :feedback "specific revision feedback"}
```

### `route`

```clojure
(k/route source routes & {:keys [default]})
```

With a route map, `source` replies with EDN `{:route key}` and the selected
workflow receives the original input.

```clojure
(k/route classifier
         {:launch launch-brief
          :incident incident-plan}
         :default launch-brief)
```

### `continue`

```clojure
(k/continue source continuation & {:keys [prompt]})
```

The continuation receives the source result text.

```clojure
(k/continue triage-agent specialist-agent)
```

### `dynamic`

```clojure
(k/dynamic agent & {:keys [max-steps]})
```

Builds a workflow whose agent emits EDN ops while the run is in progress.
Run it with the same `run` function as any other workflow:

```clojure
(k/run runner
       (k/dynamic workflow-agent :max-steps 25)
       "audit this migration")
```

Supported ops are:

```clojure
{:op :define
 :agent {:name "reader"
         :instructions "..."
         :runner :claude
         :model "sonnet"
         :tools ["Read"]
         :config {}}}
{:op :define :name "review" :workflow WORKFLOW}
{:op :patch :agent "reader" :merge {:instructions "..."}}
{:op :patch :workflow "review" :merge MAP}
{:op :remove :agent "reader"}
{:op :remove :workflow "review"}
{:op :call :agent "reader" :input "..."}
{:op :call :workflow "review" :input "..."}
{:op :spawn :agent "reader" :inputs ["a" "b"]}
{:op :complete :text "final answer"}
```

Use `agent-ref` and `workflow-ref` inside workflows defined during a dynamic
run when later patches should affect later calls.

```clojure
(k/agent-ref "reader")
(k/workflow-ref "review")
```

### `workflow?` and `defworkflow`

```clojure
(k/workflow? value)
(k/defworkflow symbol workflow)
```

Use these when storing, generating, or publishing workflows from application
code.

## Schemas

### `edn-schema` and `json-schema`

```clojure
k/edn-schema
k/json-schema
```

These are plain data reference values for the public workflow shapes. Use
`edn-schema` when documenting or generating EDN workflows, and `json-schema`
for the CLI JSON bridge.

```clojure
(keys k/edn-schema)
;=> (:agent :result :workflow :pipe :branch :delegate ...)

(get k/json-schema "$ref")
;=> "#/$defs/workflow"
```

## Rewrites

Workflow rewrites operate on EDN before execution. They never call runners and
never use `eval`.

### `agents`

```clojure
(k/agents workflow)
```

Returns the agent leaves in traversal order.

### `over`

```clojure
(k/over pred f workflow)
```

Applies `f` to every value in `workflow` where `pred` returns true, then
validates the rewritten workflow.

```clojure
(k/over k/agent?
        #(update % :instructions str "\n\nState assumptions before final answer.")
        workflow)
```

### `configure`

```clojure
(k/configure opts workflow)
```

Applies common agent configuration in one pass. Supported keys:

| Key | Meaning |
| --- | --- |
| `:runner` | Runner registry key. |
| `:model` | Model selector string. |
| `:instructions/suffix` | Text appended to every agent instruction. |

```clojure
(def production-workflow
  (->> workflow
       (k/configure {:runner :claude
                     :model "claude-sonnet-4"
                     :instructions/suffix "State assumptions before final answer."})))
```

## Execution

### `run`

```clojure
(k/run runner workflow input)
(k/run runner workflow input opts)
```

Interprets workflow data through a runner or runner registry.

```clojure
(k/run (k/mock-runner)
       (k/pipe researcher summarizer)
       "map the agent framework space")
```

Returns a result map:

```clojure
{:karcarthy/type :result
 :ok? true
 :agent "summarizer"
 :text "..."}
```

Top-level `run` throws for malformed input. Composite nodes convert child
exceptions into not-ok result maps.

### Observability

Pass `:observe` to receive OpenTelemetry-compatible span events as plain data:

```clojure
(k/run (k/mock-runner)
       (k/pipe researcher summarizer)
       "map the agent framework space"
       {:observe (fn [event] ...)})
```

Events contain span ids, parent span ids, names, attributes, paths, status, and
duration metadata. They do not include prompt text or result text.

```clojure
{:karcarthy/type :event
 :event :start                  ; :start, :finish, or :error
 :kind :workflow                ; or :agent
 :name "karcarthy.workflow.pipe"
 :span/kind :internal
 :span/id "..."
 :parent/span-id "..."
 :path [:steps 0]
 :attributes {"karcarthy.kind" "workflow"
              "karcarthy.workflow.type" "pipe"
              "karcarthy.path" "[:steps 0]"}}
```

For OpenTelemetry, start a span on `:start`, end it on `:finish` or `:error`,
use `:name` as the span name, and attach `:attributes`. karcarthy does not take
an OpenTelemetry dependency.

## Runners

### `mock-runner`

```clojure
(k/mock-runner)
(k/mock-runner respond-fn)
```

Offline deterministic runner for tests and examples.

```clojure
(k/mock-runner
 (fn [{:keys [agent prompt]}]
   (str "[" (:name agent) "] " prompt)))
```

### `fn-runner`

```clojure
(k/fn-runner f)
(k/fn-runner f opts)
```

Wraps a Clojure function as a runner for agent specs. By default, the function
receives only the input string. With `{:context? true}`, it receives
`{:agent agent :input input :opts opts}`.

```clojure
(k/fn-runner clojure.string/upper-case)

(k/fn-runner
 (fn [{:keys [agent input]}]
   (str (:name agent) ": " input))
 {:context? true})
```

### `claude-cli-runner`

```clojure
(k/claude-cli-runner)
(k/claude-cli-runner default-opts)
```

Runs agent calls through the Claude CLI. Common options include model, working
directory, turn limits, subagents, and extra Claude CLI args.

```clojure
(k/claude-cli-runner {:model "haiku"
                      :max-turns 4
                      :subagents [security-reviewer]})
```

`:subagents` is a vector of `k/subagent` values lowered to Claude Code
`--agents` JSON.

### `acp-runner`

```clojure
(k/acp-runner {:command ["some-acp-agent"]})
(k/acp-runner command opts)
```

Runs one Karcarthy agent call through an Agent Client Protocol subprocess over
stdio JSON-RPC. The ACP agent is the external process; Karcarthy folds the
agent spec's instructions into the prompt turn.

```clojure
(def acp
  (k/acp-runner {:command ["my-acp-agent"]
                 :cwd "."
                 :permission-policy :reject}))

(k/run acp
       (k/agent {:name "reviewer"
                 :instructions "Review this diff."
                 :model "sonnet"})
       "Check the current branch.")
```

Common options:

| Option | Meaning |
| --- | --- |
| `:command` | argv vector for the ACP agent process. |
| `:cwd` / `:dir` | working directory for `session/new`. |
| `:mcp-servers` | ACP MCP server specs passed during session setup. |
| `:client-capabilities` | ACP client capabilities to advertise. Defaults to none. |
| `:config-options` | session config option id/category to value map. |
| `:strict-config?` | throw if a requested config option is unavailable. |
| `:permission-policy` | `:reject` default, `:allow`, or `:cancel`. |
| `:permission-handler` | fn for ACP `session/request_permission`. |
| `:request-handler` | fn for custom agent-to-client JSON-RPC requests. |
| `:on-event` | fn called with each ACP message received. |
| `:timeout-ms` | max wait per ACP message. |

### `process-runner`

```clojure
(k/process-runner command)
(k/process-runner command opts)
```

Wraps a fixed local command. An argv vector executes directly; a string executes
through the configured shell. The command receives input on stdin and stdout
becomes result text.

```clojure
(k/process-runner ["tr" "a-z" "A-Z"])
(k/process-runner "tr a-z A-Z")
(k/process-runner ["ollama" "run" "llama3.2"]
                  {:timeout-ms 120000})
```

Options:

| Option | Meaning |
| --- | --- |
| `:trim?` | Trim stdout; defaults to true. |
| `:env` | Extra environment variables. |
| `:dir` | Working directory. |
| `:timeout-ms` | Kill the command after this many milliseconds. |
| `:shell` | argv prefix used for string commands; defaults to `["sh" "-lc"]`. |

`process-runner` does not accept a function of `k/agent`. Use the runner
registry when different agents need different fixed commands.

### `openai-agents-runner`

```clojure
(k/openai-agents-runner)
(k/openai-agents-runner default-opts)
```

Delegates single-agent calls to the OpenAI Agents SDK bridge. `:subagents` is a
vector of `k/subagent` values lowered to Agents SDK handoffs.

```clojure
(k/openai-agents-runner {:python-bin "python3"
                         :timeout-ms 120000
                         :subagents [refund-agent faq-agent]})
```

### Runner registries

```clojure
(def runners
  {:default (k/mock-runner)
   :claude  (k/claude-cli-runner {})
   :acp     (k/acp-runner {:command ["my-acp-agent"]})})

(k/run runners
       (k/pipe
        (k/agent {:name "planner"
                  :instructions "Plan."
                  :runner :claude})
        (k/agent {:name "checker"
                  :instructions "Check."
                  :runner :acp}))
       "ship the docs")
```

## Parsing generated data

```clojure
(k/read-workflow text)
(k/read-agent text)
(k/dsl-reference)
```

`read-workflow` and `read-agent` parse EDN emitted by an agent without using
`eval`.

## Results

```clojure
(k/result map)
(k/ok? result)
```

`result` fills in the standard result tag and defaults `:ok?` to true.

```clojure
(k/result {:agent "reviewer" :text "ready"})
;=> {:karcarthy/type :result
;    :ok? true
;    :agent "reviewer"
;    :text "ready"}
```

Composite workflows may add `:results`, `:subtasks`, `:rounds`, `:accepted?`,
or `:label`.
