title: Library API
description: Macros and functions for the native Clojure agent harness.
---

The public API is intentionally small.

## Agents

### `agent`

```clojure
(k/agent)
(k/agent config)
(k/agent config [rt input] body ...)
```

With no arguments, returns the model-facing `agent` Tool: the recursive
homoiconic capability. Its input is `{:source <one (agent ...) form> :input
<value>}`; it reads, expands, checks, evaluates, and invokes the resulting
Agent in the current run tree.

With a config, constructs an Agent and retains the original and macroexpanded
forms. With no body, the Agent uses the harness's standard model/tool loop.
With a body, the body is the Agent's program and must return a value satisfying
`:output`.

Required configuration:

- `:name` when using `agent` directly;
- `:model` and `:instructions` when using the default loop.

Optional configuration includes `:description`, `:context`, `:input`, `:tools`,
`:output`, `:loop`, `:memory`, `:guardrails`, `:limits`, and `:hooks`.

### `defagent`

```clojure
(k/defagent name config)
(k/defagent name config [rt input] body ...)
```

Defines a var containing an Agent. The symbol supplies the default `:name`.

### `agent?`

```clojure
(k/agent? value)
```

Returns true when the value satisfies the Agent contract.

### `source-form` and `expanded-form`

```clojure
(k/source-form agent)
(k/expanded-form agent)
```

Return the form captured before and after macroexpansion.

## Tools

### `tool`

```clojure
(k/tool config [rt input] body ...)
```

Constructs a Tool. The config requires `:name`, `:description`, and `:input`.
`:output`, `:approval`, `:enabled?`, `:guardrails`, `:timeout-ms`, `:retry`, and
`:to-model-output` are optional.

### `deftool`

```clojure
(k/deftool name config [rt input] body ...)
```

Defines a var containing a Tool. The symbol supplies the default model-facing
name.

### `tool?`

```clojure
(k/tool? value)
```

Returns true when the value satisfies the Tool contract.

### `hosted-tool` and `openai-web-search`

```clojure
(k/hosted-tool :openai {:type "web_search"})
(k/openai-web-search)
(k/openai-web-search {:search_context_size "low"})
```

A hosted Tool is executed by its matching provider rather than by karcarthy.
The model/tool loop rejects a provider mismatch. `hosted-tool?` identifies this
value type.

### `as-tool`

```clojure
(k/as-tool agent)
(k/as-tool agent options)
```

Projects an Agent as a Tool. Options may override its tool name, description,
input contract, output contract, approval policy, and context projection. The
parent remains active after the child returns.

## Execution

### `run!`

```clojure
(k/run! agent input)
(k/run! agent input options)
```

Creates a root Runtime, validates input and context, executes the Agent, and
returns a Run record.

Options:

```clojure
{:context {...}
 :limits {...}
 :memory memory
 :observe event-fn
 :approval approval-handler
 :cancel cancellation-token
 :metadata {...}}
```

### `invoke!`

```clojure
(k/invoke! rt agent input)
(k/invoke! rt agent input options)
```

Runs a child Agent inside an existing Runtime tree and returns its typed output.
The child remains visible through its lineage and lifecycle events in the root
Run trace.

### `spawn!`

```clojure
(k/spawn! rt agent input)
(k/spawn! rt agent input options)
```

Starts a child invocation under structured concurrency and returns a child
handle. The child inherits cancellation and receives a budget grant from the
parent.

### `await!` and `await-all!`

```clojure
(k/await! child)
(k/await-all! children)
```

Wait for one or several child handles. Failure and cancellation propagate as
structured exceptions unless the program catches them.

### `handoff!`

```clojure
(k/handoff! rt agent input)
```

Records an `:agent/handoff` event and invokes another Agent while retaining
runtime, budget, and trace lineage. Durable conversation transfer remains
roadmap work.

### `context`

```clojure
(k/context rt)
```

Returns the local dependency-injection value. It is not automatically exposed
to the model.

## Generated code

### `read-agent-form`

```clojure
(k/read-agent-form source)
```

Reads one Clojure form from a source string with reader evaluation disabled.
Returns the form as data.

### `check-agent-form!`

```clojure
(k/check-agent-form! rt form)
```

Fully macroexpands executable positions in the evaluation namespace while
preserving quoted source as data, then returns the checked
representation or throws a structured expansion exception. Compiler symbol and
arity failures are reported by `eval-agent-form!`.

### `eval-agent-form!`

```clojure
(k/eval-agent-form! rt checked-form)
```

Evaluates a checked form in the Runtime's evaluation namespace and verifies
that the result is an Agent.

### `compile-agent!`

```clojure
(k/compile-agent! rt source)
```

Convenience composition of `read-agent-form`, `check-agent-form!`, and
`eval-agent-form!`. Emits source, expanded-form, analysis, evaluation, and
failure events.

## Model access

### `model!`

```clojure
(k/model! rt request)
```

Low-level access to the active model transport for custom Agent programs. Most
agents should use the standard loop or invoke a model-backed child instead.

Provider transports implement only the normalized model request boundary. They
never execute tools or child agents.

## Workspace capabilities

### `workspace-tools` and `workspace-prompt`

```clojure
(def tools (k/workspace-tools {:cwd "/workspace"}))
(def instructions
  (k/workspace-prompt {:cwd "/workspace" :tools tools}))

(k/agent {:name "workspace-agent"
          :model {:provider :openai :id "gpt-5.6"}
          :instructions instructions
          :tools tools
          :output string?})
```

`workspace-tools` returns `read`, `write`, `edit`, `bash`, and `search` rooted
at an existing directory. Its options configure approval and output/time
bounds. `workspace-prompt` describes whatever local, hosted, or MCP Tools the
caller supplies. Neither function constructs a special Agent type.

## MCP

### `connect-mcp!`, `mcp-tools`, and `close-mcp!`

```clojure
(def connection
  (k/connect-mcp! {:name "server"
                   :command "server-command"
                   :args ["--stdio"]
                   :cwd "/workspace"}))
(def tools (k/mcp-tools connection))
(k/close-mcp! connection)
```

The connection implements MCP `2025-11-25` over stdio. `mcp-tools` accepts
`:prefix` and `:approval`; approval defaults to `:always`. Always close the
connection. Commands are executed directly rather than through a shell. ACP
session MCP processes start in the session `cwd`.

## ACP

### `serve-acp!`

```clojure
(k/serve-acp! {:agent agent-or-factory})
```

Serves ACP v1 JSON-RPC over `System/in` and `System/out`, or `:in` / `:out`
streams supplied in the options. A factory receives `{:cwd ... :mcp-tools ...
:session-id ...}` and returns an Agent. `:run-options` supplies base Run
options; `:client-request-timeout-ms` bounds permission requests.

The executable entry point resolves a var and serves its Agent value or
factory:

```bash
clojure -M -m karcarthy.acp your.namespace/agent-var
```

## Observation

### `emit!`

```clojure
(k/emit! rt event)
```

Adds runtime identity and lineage to an event, appends it to the Run trace, and
passes it to the configured observer.

### `events`

```clojure
(k/events run)
```

Returns the recorded observation events from a Run or Runtime.

## Contracts

Agent, tool, context, input, and output contracts initially use
`clojure.spec`. A contract adapter may support other Clojure schema systems
later, but the harness exposes one validation result shape.

See [Contracts](/docs/reference/schema) for exact values and failure data.
