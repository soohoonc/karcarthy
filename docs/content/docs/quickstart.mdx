This guide builds one model-backed Agent and runs it from a Clojure REPL.

## Requirements

- JDK 21 or newer
- Clojure CLI
- an API key for the model endpoint you use

Add karcarthy to `deps.edn`:

```clojure
{:deps
 {io.github.soohoonc/karcarthy
  {:git/url "https://github.com/soohoonc/karcarthy"
   :git/sha "<commit-sha>"}}}
 :paths ["src"]}
```

The built-in Responses transport reads `OPENAI_API_KEY` by default.

## Define a Tool

Create `src/example.clj`:

```clojure
(ns example
  (:require [karcarthy :as k]))

(k/deftool word-count
  {:description "Count the words in a string."
   :input {:type "object"
           :properties {:text {:type "string"}}
           :required ["text"]
           :additionalProperties false}
   :output integer?}
  [_ {:keys [text]}]
  (count (re-seq #"\S+" text)))
```

`deftool` defines a var containing a Tool. Its input is validated before
execution, and its output is validated before being returned to the model.

## Define an Agent

Add this to the same namespace:

```clojure
(k/defagent editor
  {:model {:transport :responses
           :id "gpt-5.6"}
   :instructions
   "Edit the supplied text. Use word-count when the user asks about length."
   :tools [word-count]
   :output string?})
```

An Agent without a Clojure body uses karcarthy's model/Tool loop. `defagent`
defines a named var and supplies the default Agent name (`"editor"` here).

Use `agent` when you want an Agent value as an expression:

```clojure
(def concise-editor
  (k/agent
   {:name "concise-editor"
    :model {:transport :responses :id "gpt-5.6"}
    :instructions "Rewrite the input in at most 50 words."
    :output string?}))
```

Both forms create the same kind of Agent.

## Run from the REPL

Start a REPL in the project directory:

```bash
clojure -M
```

```clojure
(require '[example :as example] :reload)

(def run
  (k/run! example/editor
          "Shorten this paragraph and report its original word count: ..."))

(select-keys run [:status :output :usage])
(k/events run)
```

After editing `example.clj`, evaluate the changed form or reload the namespace
and run it again. See the [REPL example](/docs/examples/repl) for an entirely
offline setup using `fake-model`.

## Enable recursive Agent creation

Zero-arity `(k/agent)` returns a Tool named `agent`. Add it when the model
should be able to define and run a specialized child:

```clojure
(k/defagent architect
  {:model {:transport :responses :id "gpt-5.6"}
   :instructions
   (str "Complete the task. When a specialist would help, call the agent "
        "Tool with a focused Agent definition and its input.")
   :tools [word-count (k/agent)]
   :output string?})

(k/run! architect "Analyze and improve this document: ..."
        {:limits {:agent-depth 4
                  :generated-forms 8
                  :model-calls 30}})
```

The Tool accepts one `(agent ...)` source form plus its input. The generated
Agent is checked, evaluated, and invoked as a child of the current Run. The
[Agents guide](/docs/agents) covers known Agent Tools, generated Agents,
recursion, and source inspection.

## Add conversation history

Runs are independent unless you pass a Session:

```clojure
(def session (k/memory-session))

(k/run! editor "Remember that the target length is 80 words."
        {:session session})

(k/run! editor "Now revise the draft I sent earlier."
        {:session session})
```

`memory-session` is process-local. Applications can implement the small
`karcarthy.session/Session` protocol with their own durable store.
