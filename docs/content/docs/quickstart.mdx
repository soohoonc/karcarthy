Start here. Every run begins with one request map:

```clojure
(k/run {:runner runner
        :workflow workflow
        :input input
        :options options})
```

`workflow` can be a single agent or a composed workflow. `runner` is the value
that runs each agent call. `input` is the initial value given to the workflow.
`options` is runtime configuration such as observation callbacks.

| Key | Meaning |
| --- | --- |
| `:runner` | The backend used for each agent call, such as mock, ACP, Claude, Codex, OpenAI, a process, or a local function. |
| `:workflow` | The agent or workflow data to run. |
| `:input` | The initial workflow value. A string is a prompt; EDN is rendered before the first agent call. |
| `:options` | Runtime controls for this run, such as `:observe`, `:state`, timeouts, or runner-specific overrides. |

## Input Shape

The common case is a string prompt:

```clojure
(k/run {:runner runner
        :workflow workflow
        :input "review this change"})
```

Use a map when the initial task has structure:

```clojure
(k/run {:runner runner
        :workflow workflow
        :input {:prompt "review this change"
                :ticket 42
                :risk :high}})
```

That starts the first agent with:

```text
review this change

INPUT EDN:
{:ticket 42, :risk :high}
```

A map without `:prompt` is passed as EDN text. Put execution configuration in
`:options`, runner construction, or agent fields such as `:model` and `:tools`,
not inside `:input`.

## Requirements

- JDK 21 or newer
- Clojure CLI
- A checkout of `soohoonc/karcarthy`

## Run One Agent

```clojure
(require '[karcarthy :as k])

(k/defagent echo
  {:instructions "Echo the input."})

(k/run {:runner (k/mock-runner)
        :workflow echo
        :input "hi"})
```

Result:

```clojure
{:karcarthy/type :result
 :ok? true
 :agent "echo"
 :text "[echo] hi"
 :raw {:runner :mock}}
```

That is the smallest workflow: one request, one agent node, one runner.

## Compose Two Agents

```clojure
(k/defagent researcher
  {:instructions "Research the question and list the important facts."})

(k/defagent summarizer
  {:instructions "Summarize the previous answer in one sentence."})

(def workflow
  (k/pipe researcher summarizer))

(k/run {:runner (k/mock-runner)
        :workflow workflow
        :input "why does Lisp use code as data?"})
```

`pipe` feeds the first result's `:text` into the second agent.

## Choose The Runner At Runtime

Keep agents runner-agnostic:

```clojure
(k/defagent reviewer
  {:instructions "Review the input and return concrete issues."
   :model "sonnet"})
```

Pick the runner before calling `run`:

```clojure
(defn review [runner input]
  (k/run {:runner runner
          :workflow reviewer
          :input input}))

(review (k/mock-runner) "check this design")
(review (k/claude-runner {}) "check this design")
(review (k/codex-runner {:dir "."}) "check this design")
(review (k/acp-runner {:command ["my-acp-agent"]}) "check this design")
```

Agents do not carry a backend. If the runner changes at runtime, pass a
different runner value.

## CLI

Build the CLI jar:

```bash
clojure -T:build uber
```

Run one mock agent:

```bash
./bin/karcarthy agent echo --instructions "Echo the input." hi
```

Expected output:

```text
[echo] hi
```

Choose the CLI runner for the whole invocation with `--runner`:

```bash
./bin/karcarthy agent reviewer \
  --runner claude \
  --instructions "Review the input and return concrete issues." \
  "check this design"
```
