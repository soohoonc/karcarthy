## Install

karcarthy requires JDK 21 and the Clojure CLI. Add it to `deps.edn`:

```clojure
{:deps
 {io.github.soohoonc/karcarthy
  {:git/url "https://github.com/soohoonc/karcarthy"
   :git/sha "<commit-sha>"}}
 :paths ["src"]}
```

The Responses transport reads `RESPONSES_API_KEY` or `OPENAI_API_KEY`.

## Define a Tool

Create `src/example.clj`:

```clojure
(ns example
  (:require [karcarthy :as k]))

(k/deftool word-count
  {:description "Count the words in text."
   :input {:type "object"
           :properties {:text {:type "string"}}
           :required ["text"]
           :additionalProperties false}
   :output integer?}
  [{:keys [text]}]
  (count (re-seq #"\S+" text)))
```

A Tool body receives one input value. Its schema tells the model what to send;
karcarthy validates the input before the call and the output afterward.

## Define an Agent

Add this to the same file:

```clojure
(k/defagent editor
  {:model {:transport :responses
           :id "gpt-5.6"}
   :instructions
   "Edit the user's text. Use word-count when the user asks about length."
   :tools [word-count]
   :output string?})
```

An Agent without a Clojure body uses karcarthy's model and Tool loop.
`defagent` defines a var containing that Agent.

## Run it

Start a REPL in the project directory:

```bash
clojure -M
```

```clojure
(require '[karcarthy :as k]
         '[example :as example] :reload)

(def run
  (k/run! example/editor
          "Shorten this paragraph and report its original word count: ..."))

(select-keys run [:status :output :usage])
(k/events run)
```

`run!` returns a Run even when execution fails or is cancelled. Check
`:status` before using `:output`.

After changing `example.clj`, reload the namespace and run the Agent again.
