---
title: Programs
description: Express sequencing, branching, parallelism, recursion, and generated agents as ordinary Clojure.
---

karcarthy programs are Clojure programs. The library supplies agent-specific
effects and observation; it does not introduce a workflow AST or interpreter.

## Sequential work

Use `let` when one result feeds another agent:

```clojure
(k/defagent research-then-write
  {:input ::question
   :output ::article}
  [rt question]
  (let [research (k/invoke! rt researcher question)
        outline  (k/invoke! rt outliner research)]
    (k/invoke! rt writer {:question question
                          :research research
                          :outline outline})))
```

Every `invoke!` creates a child execution and enforces the child's input,
context, output, budget, and guardrail contracts.

## Branching and routing

Use the host language:

```clojure
(case (:kind request)
  :incident (k/invoke! rt incident-agent request)
  :launch   (k/invoke! rt launch-agent request)
  (k/invoke! rt general-agent request))
```

The decision may come from normal data, a tool result, structured model output,
or another agent. No model-specific routing protocol is built into the
harness.

## Parallel work

`spawn!` creates a child agent execution and `await-all!` joins children while
preserving cancellation, budgets, and event lineage:

```clojure
(let [jobs (for [reviewer reviewers]
             (k/spawn! rt reviewer change))
      reviews (k/await-all! jobs)]
  (k/invoke! rt editor {:change change :reviews reviews}))
```

Parallelism is bounded by runtime policy:

```clojure
(k/run! review-team change
        {:limits {:parallelism 4
                  :agent-depth 6
                  :model-calls 80}})
```

## Iteration and revision

Use `loop/recur` for feedback cycles:

```clojure
(loop [draft (k/invoke! rt writer request)
       round 1]
  (let [{:keys [accepted? feedback]}
        (k/invoke! rt critic draft)]
    (cond
      accepted? draft
      (= round 3) draft
      :else (recur (k/invoke! rt writer
                              {:request request
                               :draft draft
                               :feedback feedback})
                    (inc round)))))
```

The application decides whether an unaccepted final draft is usable. That
policy is visible in code rather than hidden in node semantics.

## Agents as tools

Expose a child agent to a model-backed parent with `as-tool`:

```clojure
(k/defagent support-manager
  {:model {:transport :responses :id "gpt-5.6"}
   :instructions "Help the user. Delegate specialized work when useful."
   :tools [(k/as-tool billing-agent
                      {:name "billing"
                       :description "Handle billing questions."})
           (k/as-tool refund-agent
                      {:name "refund"
                       :description "Evaluate refund requests."})]})
```

The manager retains control. karcarthy does not expose a handoff operation:
`invoke!` and `as-tool` both return control to the caller. A future handoff API
must first define real control and Session-history transfer semantics.

## Generated programs

Give a model-backed Agent `(k/agent)` in its tool set when it should be able to
produce and run source like this:

```clojure
(agent
  {:name "adaptive-review"
   :input ::change
   :output ::review}
  [rt change]
  (let [risk (invoke! rt risk-classifier change)]
    (if (= :high (:level risk))
      (let [reviews (await-all!
                     [(spawn! rt security-reviewer change)
                      (spawn! rt reliability-reviewer change)])]
        (invoke! rt editor {:change change :reviews reviews}))
      (invoke! rt general-reviewer change))))
```

The compilation path is:

```text
source → read → macroexpand → contracts → eval → Agent → invoke!
```

The evaluated child may itself receive `(agent)` and recursively create more
children. Depth and resource limits belong to the runtime, not to a special
language mode.

## Structural rewriting

Because source is a Clojure form, transformations use `clojure.walk` or macros:

```clojure
(defn replace-model [form model]
  (clojure.walk/postwalk
   (fn [x]
     (if (and (map? x) (contains? x :model))
       (assoc x :model model)
       x))
   form))
```

A search procedure may quote an existing program, mutate it, compile the
candidate, and evaluate its behavior. The stored source and expanded forms
make the transformation auditable.

## Macros as reusable patterns

Repeated patterns may become normal macros:

```clojure
(defmacro with-review [[draft binding] & body]
  `(let [~binding (k/invoke! rt critic ~draft)]
     ~@body))
```

Macros extend Clojure rather than adding workflow node kinds. Model-authored
code can use the same macros available in its evaluation namespace.

## Failure, cancellation, and suspension

Agent effects throw structured exceptions inside a custom program. Use normal
`try`/`catch` when the program can recover. Runtime cancellation propagates to
children, model calls, and tools. Approval is checked before tool execution;
durable suspension and resumption remain roadmap work.
