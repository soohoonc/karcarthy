title: Limits and safety
description: Bound a run and understand the authority of eval and Tools.
---

```clojure
(k/run! agent input
        {:limits {:model-calls 40
                  :input-tokens 500000
                  :output-tokens 50000
                  :depth 4
                  :concurrency 4
                  :evals 8
                  :deadline-ms 300000}})
```

All `run!` calls made during the first call's dynamic extent share these
budgets. Agent-specific and participating-call limits may narrow them, never
widen them.

| Limit | Bounds |
| --- | --- |
| `:model-calls` | Attempted model turns |
| `:input-tokens` / `:output-tokens` | Reported model usage |
| `:depth` | Participating Agent call depth |
| `:concurrency` | Concurrent Tool and dynamically started Agent work |
| `:evals` | Eval attempts, including failed attempts |
| `:deadline-ms` | Wall-clock time from the first `run!` |

## Cancellation and approvals

Pass a function, atom, or dereferenceable value through `:cancel`. Cancellation
is checked while waiting for model, Tool, and participating Agent work. It
interrupts in-flight Java futures and produces a cancelled Run with lifecycle
events retained. Third-party transports and Tools should respond to thread
interruption and close their own resources; the built-in process Tools terminate
their child process.

Tools may request approval on every call or once per run. Without an approval
handler, required approval fails closed. Approval controls Tool calls; it does
not reduce the operating-system authority of the implementing process.

## Eval is trusted code

karcarthy reads exactly one expression with reader evaluation disabled. It then
evaluates that expression as JVM Clojure. The single-expression rule creates a
clear protocol boundary; it is not a security boundary. `(do ...)` can contain
arbitrary operations.

Use `eval` only for code trusted with the host JVM's permissions. For untrusted
generation, isolate the entire application process and restrict filesystem,
network, credentials, and namespaces. MCP servers and shell Tools require the
same treatment.
