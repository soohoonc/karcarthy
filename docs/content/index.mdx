> **Agents write Clojure that runs Agents.**

karcarthy is a small Clojure agent library built from Agents, Tools, Runs,
Sessions, context, limits, and events. Every Agent also receives an `eval` Tool
that evaluates one ordinary Clojure expression.

```clojure
(k/defagent assistant
  {:model "gpt-5.6"
   :instructions "Answer clearly."
   :output string?})

(k/run! assistant "Why are Lisp macros unusual?")
```

## Dynamic workflows use the language

An Agent can write:

```clojure
(let [reviewer (agent {:name "reviewer"
                       :model "gpt-5.6"
                       :instructions "Find one dangerous assumption."
                       :input string?
                       :output string?})
      jobs (mapv #(future (run! reviewer %)) input)]
  (mapv (comp :output deref) jobs))
```

karcarthy reads one expression with reader evaluation disabled, expands its
macros, and evaluates it in the same JVM. The expression can branch, recurse,
create Agents and Tools, run Agents concurrently, and return JSON-compatible
data to the model.

There is no workflow graph between the generated code and Clojure.

## One dynamically scoped run

The first `run!` establishes a run. Calls made during its dynamic extent join
that run, including calls inside Clojure futures. They share limits, usage,
events, cancellation, approvals, context, and the run ID. Each Agent gets a
fresh model conversation.

```clojure
(->> (k/events result)
     (filter #(= :eval/expanded (:type %)))
     (map :expression))
```

## Known workflows are the same Clojure

```clojure
(defn review-team [change]
  (->> [security-reviewer api-reviewer]
       (mapv #(future (k/run! % change)))
       (mapv deref)
       (mapv :output)))
```
