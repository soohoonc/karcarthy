> **Agents write Clojure that runs Agents.**

karcarthy is a small, homoiconic agent library for Clojure. It owns the model
and Tool loop while leaving orchestration in the host language.

An Agent can answer directly, call a Tool, delegate to an available Agent, or
evaluate one ordinary Clojure expression that creates and runs the Agents its
task needs. There is no workflow graph or second orchestration language between
generated code and Clojure.

## Run one Agent

```clojure
(require '[karcarthy :as k])

(k/defagent reviewer
  {:model "gpt-5.6"
   :instructions "Review the proposed change. Report only concrete defects."
   :input-schema string?
   :output-schema string?})

(def result
  (k/run! reviewer
          "Contract: count may be zero. Change: average returns total / count."))

(:status result) ;=> :completed
(:output result) ;=> "..."
```

Agents and Tools are flat Clojure data. `run!` returns a Run map with status,
output, usage, events, and a structured error when execution fails.

## Orchestrate with Clojure

When the application already knows the participants, write an ordinary
function:

```clojure
(->> [correctness-reviewer concurrency-reviewer]
     (mapv #(future (k/run! % change)))
     (mapv (comp k/output deref)))
```

When the right participants depend on the task, an Agent can generate the same
kind of Clojure after it reads the input:

```clojure
(let [reviewer
      (agent {:name "concurrency-reviewer"
              :model "gpt-5.6"
              :instructions "Check whether two workers can claim one job."
              :input-schema string?
              :output-schema string?})]
  (output (run! reviewer input)))
```

That expression may use normal control flow, functions, macros, concurrency,
recursion, Agent construction, Tool construction, and `run!`. An Agent it
creates receives the same capability, so delegation can be nested.

## One Run follows the code

The first `run!` establishes a Run. Calls in its dynamic extent—including calls
inside `future`—share the Run ID, limits, usage, events, cancellation,
approvals, context, and executor. Each participating Agent still starts a fresh
model conversation with only its explicit input.
