> **The agent architecture is the program.**

karcarthy is an experiment in writing agents as Clojure programs. Developers
and models create the same Agent forms, and the same runtime executes them.

Most agent frameworks introduce another representation for orchestration: a
graph, a collection of nodes, or a JSON workflow. In karcarthy, fixed
architecture is ordinary Clojure. When the right architecture depends on the
task, a running Agent can write another Clojure Agent.

## The same program

A developer can write an Agent:

```clojure
(k/agent
 {:name "reviewer"
  :model {:transport :responses :id "gpt-5.6"}
  :instructions "Review the proposed change."
  :input string?
  :output string?})
```

A model can write the same form during a Run:

```clojure
(agent
 {:name "reviewer"
  :model {:transport :responses :id "gpt-5.6"}
  :instructions "Review the proposed change."
  :input string?
  :output string?})
```

Both produce the same Agent value:

```text
developer ─┐
           ├─ Agent form → read → expand → check → evaluate → run
model ─────┘
```

The same contracts, limits, event stream, and model/Tool loop apply. That is
the reason karcarthy is written in Clojure: the program is already data that a
developer, macro, or model can produce.

## Watch it happen

The Architect example gives one Agent a task and requires it to create two
specialists. The terminal redraws their state while they run:

```text
Run run_7c2e9b… · running · 18s · 3 model calls · 8,421 tokens · 2 Agent forms
└─ architect · waiting for 2 Agents
   ├─ failure-analyst · calling model
   └─ rollout-planner · calling model
```

```bash
clojure -M:examples architect \
  "Review a migration from synchronous writes to a queue."
```

The exact names and answers vary because the run is live. The tree is not a
workflow definition: it is an event-derived view of the Clojure Agents that
are executing.

## Boundaries

karcarthy owns the bounded model/Tool loop. Clojure owns control flow. A Run
records one execution; a Session optionally carries model conversation between
Runs.

Runtime Agent source is evaluated as JVM Clojure. karcarthy verifies that the
source produces an Agent, but it is not a sandbox for untrusted code. This is a
side project exploring the programming model, not a claim that arbitrary model
output is safe to evaluate in a privileged process.
