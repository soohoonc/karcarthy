> **Agents write Clojure that runs Agents.**

karcarthy is a small Clojure agent library. Its distinctive feature is a
built-in `eval` Tool: while an Agent is running, it can write one ordinary
Clojure expression that creates Agents, runs them, and combines their results.

There is no workflow graph between generated code and Clojure.

## Start with one Agent

```clojure
(require '[karcarthy :as k])

(k/defagent assistant
  {:model "gpt-5.6"
   :instructions "Answer clearly."
   :input-schema string?
   :output-schema string?})

(def garden-report
  "A moon garden's leaves are turning silver. What should we check first?")

(k/run! assistant garden-report)
```

An Agent is a flat Clojure map. `run!` returns a Run map containing its status,
output, usage, events, and structured error when one occurs.

## Let the task choose specialists

Every Agent also receives `eval`. The Tool accepts a single `code` string. The
current Agent input is already available as `input` inside the expression.

For the garden task, the Agent might create a botanist and a radiation engineer:

```clojure
(let [botanist (agent {:name "botanist"
                       :model "gpt-5.6"
                       :instructions "Find biological causes in the evidence."
                       :input-schema string?
                       :output-schema string?})
      radiation-engineer
      (agent {:name "radiation-engineer"
              :model "gpt-5.6"
              :instructions "Find radiation and shielding causes."
              :input-schema string?
              :output-schema string?})
      jobs (mapv #(future (run! % input))
                 [botanist radiation-engineer])]
  (mapv (comp :output deref) jobs))
```

karcarthy reads exactly one expression with reader evaluation disabled,
macroexpands it, and evaluates it in the same JVM. The expression can branch,
recurse, create Agents and Tools, run work concurrently, and return a model-safe
value to the calling Agent.

## Keep one Run

The first `run!` establishes a Run. Calls made during its dynamic extent join
that Run, including calls inside Clojure futures. They share limits, usage,
events, cancellation, approvals, context, and the Run ID. Each Agent gets a
fresh model conversation.

Known workflows use the same primitives. If the team is part of the application,
write an ordinary Clojure function instead of asking a model to generate it.
