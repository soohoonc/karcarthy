> **The agent architecture is the program.**

karcarthy is a toy agent harness built around Clojure forms and macros.

An Agent is constructed with `(agent ...)`. During a Run, the model can call a
built-in Tool with the source of another `(agent ...)` form. karcarthy reads the
source, expands the macro, checks the result, evaluates it, and runs the new
Agent inside the current Run.

The program adds another Agent by evaluating more Lisp while it runs. There is
no workflow graph or separate orchestration language between the generated
form and Clojure.

## Watch the program add new Agents

The Architect example starts one Agent and asks it to create two specialists:

```bash
clojure -M:examples architect \
  "Review a migration from synchronous writes to a queue."
```

The terminal follows the running program:

```text
Run run_7c2e9b… · running · 18s · 3 model calls · 8,421 tokens · 2 Agent forms
└─ architect · waiting for 2 Agents
   ├─ failure-analyst · calling model
   └─ rollout-planner · calling model
```

The parent did not choose from two predefined workers. It submitted two new
Clojure forms. Each form was evaluated into an Agent and run. Their results
returned to the parent, which used them to produce the final answer.

At the REPL, the generated forms and final answer remain available:

```clojure
(println (:output run))

(->> (k/events run)
     (filter #(= :program/expanded (:type %)))
     (map :definition))
```

## Code is data

Before evaluation, an Agent is a Clojure form:

```clojure
(agent
 {:name "failure-analyst"
  :model {:transport :responses :id "gpt-5.6"}
  :instructions "Find the most dangerous failure mode."
  :input string?
  :output string?})
```

karcarthy retains the form, expands the `agent` macro, checks the expansion,
and evaluates it. The result is an executable Agent value.

```text
form → macroexpansion → Agent value → execution
```

A model-driven Agent receives the same built-in Tool, so this can continue
recursively.

## Known architecture is ordinary Clojure

Runtime generation is not required. When the architecture is known, write it
directly:

```clojure
(k/agent
 {:name "review-team"
  :input string?
  :output vector?}
 [change]
 (->> [security-reviewer api-reviewer]
      (mapv #(future (k/run! % change)))
      (mapv deref)
      (mapv :output)))
```

Branches, functions, recursion, macros, and concurrency remain Clojure.
karcarthy does not provide workflow nodes for operations the language already
has.
