---
title: Build a coding agent
description: Give an Agent repository Tools and an open-ended task.
---

The Coding Agent can read, search, edit, and run commands inside one repository.
It inspects the project, makes focused changes, and verifies the result.

## Run it

Set `OPENAI_API_KEY`, then provide an absolute repository path and a task:

```bash
clojure -M:examples coding /workspace/project \
  "Investigate the failing integration tests, fix the cause, and verify it."
```

The command prints the Agent's final account of the cause, changes, and
verification. The answer is live model output and therefore varies; the
repository diff and test results are the evidence to inspect.

## The Agent

The example roots local Tools at the supplied directory:

```clojure
(k/agent
 {:name "coding-agent"
  :model {:transport :responses
          :provider :openai
          :id "gpt-5.6"
          :reasoning :medium}
  :instructions
  "Inspect the repository and run its tests before changing it. Diagnose the
   cause, make focused edits, and run the relevant tests again."
  :tools (k/local-tools {:cwd cwd})
  :input-schema string?
  :output-schema string?
  :max-turns 24})
```

`local-tools` supplies `read`, `write`, `edit`, `bash`, and `search`. File
operations remain under `:cwd`, including after symlink resolution. Shell
execution is bounded but is not sandboxed.

The complete example also sets limits on model calls, eval, Agent depth,
concurrency, and total time. See
[`examples/coding/main.clj`](https://github.com/soohoonc/karcarthy/blob/main/examples/coding/main.clj).

## Let the task choose the architecture

The built-in `eval` Tool is present automatically. The Coding Agent may solve
the task directly or write Clojure that creates a focused reviewer when that
helps. The prompt does not require a fixed topology.

That distinction matters: [code orchestration](/docs/agent-orchestration#orchestrate-in-code)
defines participants before the task, while the [code-review example](/docs/guides/review)
creates reviewers after reading the change.

Use the [Harbor integration](/docs/protocols/harbor) to run this Agent against
an isolated repository task and a behavioral verifier.
