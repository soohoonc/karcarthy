---
title: Agent composition
description: Coordinate independent Agent Runs with ordinary Clojure.
---

This example runs two reviewers in parallel and passes their outputs to an
editor. Each `run!` call is independent, so the example checks every Run and
sets limits for each reviewer explicitly. The complete source lives at
`examples/composition/main.clj`.

## Build the system

```clojure
(ns example.composition
  (:require [karcarthy :as k]))

(defn output! [run]
  (if (= :completed (:status run))
    (:output run)
    (throw (ex-info "Agent Run failed" {:run run}))))

(defn review-system [cwd]
  (let [tools
        (k/local-tools
         {:cwd cwd
          :approval {:read :never
                     :search :never
                     :write :always
                     :edit :always
                     :bash :always}})

        reviewer-options
        {:limits {:model-calls 12
                  :deadline-ms 120000}
         :approval (constantly false)}

        security-reviewer
        (k/agent
         {:name "security-reviewer"
          :model {:transport :responses :id "gpt-5.6"}
          :instructions
          "Inspect the repository for concrete security problems. Cite files."
          :tools tools
          :input string?
          :output string?})

        api-reviewer
        (k/agent
         {:name "api-reviewer"
          :model {:transport :responses :id "gpt-5.6"}
          :instructions
          "Inspect the repository for API compatibility problems. Cite files."
          :tools tools
          :input string?
          :output string?})

        editor
        (k/agent
         {:name "review-editor"
          :model {:transport :responses :id "gpt-5.6"}
          :instructions
          "Combine the supplied reviews. Remove duplicates and unsupported claims."
          :input string?
          :output string?})]

    (k/agent
     {:name "review-team"
      :input string?
      :output string?}
     [request]
     (let [[security api]
           (->> [security-reviewer api-reviewer]
                (mapv #(future (k/run! % request reviewer-options)))
                (mapv deref)
                (mapv output!))
           editor-prompt
           (str "Request:\n" request
                "\n\nSecurity review:\n" security
                "\n\nAPI review:\n" api)]
       (output! (k/run! editor editor-prompt
                        {:limits {:model-calls 4
                                  :deadline-ms 60000}}))))))
```

The reviewers use the model and Tool loop. `review-team` uses a Clojure body
for deterministic coordination. `future` supplies concurrency; `output!`
propagates failed child Runs instead of quietly passing `nil` to the editor.

## Run it

With `OPENAI_API_KEY` set, load the example from the repository root:

```clojure
(load-file "examples/composition/main.clj")

(def team (example.composition/review-system "/workspace/project"))

(def run
  (k/run! team "Review the current working tree against origin/main."))

(select-keys run [:status :output :usage])
```

The outer Run records the coordinator. Reviewer and editor calls are separate
Runs with their own limits and events.

## Let the model choose the specialists

When the useful decomposition is not known in advance, let the model write a
new Agent instead of hard-coding the review team:

```clojure
(defn adaptive-reviewer [cwd]
  (k/agent
   {:name "adaptive-reviewer"
    :model {:transport :responses :id "gpt-5.6"}
    :instructions
    "Review the repository. Create a focused Agent when specialization helps."
    :tools (k/local-tools {:cwd cwd})
    :output string?}))

(k/run! (adaptive-reviewer "/workspace/project")
        "Review the current change."
        {:limits {:depth 4
                  :agent-forms 8
                  :parallelism 3
                  :model-calls 40}})
```

Agent forms authored at runtime remain inside this Run, so these limits and events cover the
whole Agent tree. Continue with [Coding](/docs/guides/coding)
for open-ended live repository work and optional model-authored Agents.
