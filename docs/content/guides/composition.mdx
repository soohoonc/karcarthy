---
title: Compose a known team
description: Coordinate a fixed Agent architecture with ordinary Clojure.
---

Use dynamic eval when the task should determine the architecture. When the team
is already part of your application, write it directly.

This version of the moon-garden workflow always runs a botanist and a radiation
engineer, then gives both reports to an editor.

## Write the coordinator

```clojure
(defn output! [run]
  (if (= :completed (:status run))
    (:output run)
    (throw (ex-info "Agent Run failed" {:run run}))))

(defn diagnose-garden [report]
  (let [[biology radiation]
        (->> [botanist radiation-engineer]
             (mapv #(future (k/run! % report)))
             (mapv deref)
             (mapv output!))
        editor-input
        (str "Garden report:\n" report
             "\n\nBiology:\n" biology
             "\n\nRadiation and shielding:\n" radiation)]
    (output! (k/run! editor editor-input))))
```

The coordinator is a normal function. `future` runs both reviews together, and
`output!` prevents a failed Run from being passed to the editor.

## Run it

With `OPENAI_API_KEY` set, start a REPL from the repository root:

```clojure
(load-file "examples/composition/main.clj")

(example.composition/diagnose-garden
 "The leaves turned silver two days after a solar storm; humidity is stable.")
```

Each `run!` establishes an independent Run when this function is called outside
an active Run. If evaluated code calls the same function, its Agent calls join
the active Run.

Compare this fixed team with [Let an Agent choose the team](/docs/guides/architect),
where the task determines which specialists exist.
