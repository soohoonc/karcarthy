---
title: Evaluate an agent
description: Run the Coding Agent in isolation, score it, and inspect its trajectory.
---

This example runs the karcarthy Coding Agent against a broken repository,
checks its work with hidden tests, and records the interaction for inspection.
It uses Harbor as the evaluation harness and ACP as the process boundary.

## What is measured

The bundled task gives the Agent this production symptom:

```text
A scheduler is delivering some jobs twice and retrying failed jobs in a tight
loop. Find and fix the persistence bugs without changing the public Store
method signatures. Preserve priority ordering and exhausted-attempt behavior.
```

After the Agent exits, hidden behavioral tests run against its repository:

```text
reward 1  all hidden tests pass
reward 0  one or more hidden tests fail
```

The repository does not check in a model result. Running the command below
creates the score, logs, and trajectory for the selected model.

## Before you begin

You need Docker, JDK 21, the Clojure CLI, `uv`, and an OpenAI credential. The
model-backed run is paid and requires an explicit opt-in.

## Run the evaluation

```bash
KARCARTHY_LIVE=1 OPENAI_API_KEY=... examples/harbor/run.sh
```

Set `KARCARTHY_OPENAI_MODEL` to choose another Responses-compatible model and
`KARCARTHY_ATTEMPTS` to repeat the task.

The script packages the fixed Coding Agent, starts it through ACP inside the
isolated task environment, and writes the resulting job under
`examples/harbor/jobs`.

## Inspect the result

Open Harbor's local viewer:

```bash
uvx --python 3.13 --from harbor harbor view examples/harbor/jobs
```

The job contains:

- the verifier reward and hidden-test logs;
- the Agent's final answer;
- model messages and token usage;
- repository Tool calls and observations; and
- any Agent-authored eval and participating Agent calls.

Use these artifacts together. The reward says whether the behavior passed; the
trajectory shows how the Agent reached that result.

## Validate the task without a model

The bundled oracle checks that the environment and verifier can reach reward
`1` without spending model tokens:

```bash
uvx --python 3.13 --from harbor harbor run \
  --path examples/harbor/tasks/scheduler \
  --agent oracle \
  --jobs-dir examples/harbor/jobs \
  --job-name oracle \
  --yes
```

The operational packaging details live in
[`examples/harbor/README.md`](https://github.com/soohoonc/karcarthy/blob/main/examples/harbor/README.md).
[ACP server](/docs/protocols/acp) explains the boundary Harbor launches.
