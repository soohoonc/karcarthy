[Harbor](https://www.harborframework.com/) runs Agents in isolated task
environments, invokes verifiers, and stores rewards and trajectories. The
karcarthy example exposes one fixed Coding Agent through ACP:

This is the last step in the guide sequence. Read [Coding](/docs/guides/coding)
for the Agent being evaluated and [ACP](/docs/protocols/acp) for the process
boundary Harbor launches.

```text
Harbor task
  -> packaged karcarthy ACP process
  -> fixed Coding Agent and repository Tools
  -> Harbor verifier
  -> reward, logs, and ATIF trajectory
```

This is an evaluation example. It has no optimizer, training loop, or program
search.

## What it demonstrates

1. A normal karcarthy Agent can be packaged and served through ACP.
2. Harbor can run that Agent inside an isolated repository task.
3. A task verifier can define a metric over the Agent's work.
4. Harbor can store the reward and convert the ACP interaction into an ATIF
   trajectory.

Harbor owns the task environment, verifier, result storage, and trajectory.
karcarthy owns the Agent and model/Tool loop.

## Bundled task and metric

The bundled `scheduler` task contains a small SQLite-backed delivery scheduler
with transaction and retry bugs. The Coding Agent receives a production
symptom, inspects the repository, makes changes, and runs its tests.

After the Agent exits, Harbor runs hidden behavioral tests. The verifier's
`reward` metric is:

```text
1  all hidden tests pass
0  one or more hidden tests fail
```

The task also includes an oracle solution so its environment and verifier can
be checked without a model call:

```bash
uvx --python 3.13 --from harbor harbor run \
  --path examples/harbor/tasks/scheduler \
  --agent oracle \
  --jobs-dir examples/harbor/jobs \
  --job-name oracle \
  --yes
```

## Run the live evaluation

Docker, JDK 21, the Clojure CLI, `uv`, and an OpenAI credential are required.
The model-backed run is paid and explicitly opt-in:

```bash
KARCARTHY_LIVE=1 OPENAI_API_KEY=... examples/harbor/run.sh
```

Set `KARCARTHY_OPENAI_MODEL` to select another Responses-compatible model and
`KARCARTHY_ATTEMPTS` to repeat the evaluation.

The development adapter builds and uploads a Harbor-only application plus a
Java 21 runtime. The published karcarthy library artifact remains unchanged.

## Inspect the result

Harbor stores the result, verifier logs, ACP events, and
[`trajectory.json`](https://www.harborframework.com/docs/agents/trajectory-format)
under `examples/harbor/jobs`. Open the viewer with:

```bash
uvx --python 3.13 --from harbor harbor view examples/harbor/jobs
```

The ATIF trajectory records model messages, Tool calls, observations, usage,
and any runtime `agent` calls. karcarthy's internal Run events retain finer
compiler phases and nested lineage when those details are needed.
