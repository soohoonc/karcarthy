The public examples use the live Responses transport. Fake models stay in the
offline unit suite, where they make kernel behavior deterministic and free.

Each example is an application in its own folder with a locally scoped
`main.clj`. They are not library namespaces and are not included in the
published artifact.

## Guides

| Example | Add this concept | Source |
| --- | --- | --- |
| [Basic Agent](/docs/quickstart#run-the-live-example) | One live model Run. | [`basic/main.clj`](https://github.com/soohoonc/karcarthy/blob/main/examples/basic/main.clj) |
| [Chat](/docs/guides/chat) | Conversation history across Runs. | [`chat/main.clj`](https://github.com/soohoonc/karcarthy/blob/main/examples/chat/main.clj) |

| [Coding](/docs/guides/coding) | Open-ended Tool use where the model chooses its own strategy. | [`coding/main.clj`](https://github.com/soohoonc/karcarthy/blob/main/examples/coding/main.clj) |

The progression is additive rather than a taxonomy of competing demos. The
Coding Agent is the main end-to-end application. Harbor then treats a complete
Clojure Agent form as the search artifact and evaluates replacements for that
application on isolated tasks.

The [Quickstart REPL section](/docs/quickstart#develop-at-the-repl) covers
interactive development. MCP and ACP live under [Protocols](/docs/protocols/mcp).
