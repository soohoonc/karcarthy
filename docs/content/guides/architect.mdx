---
title: Let an Agent choose the team
description: Run the central example and inspect the Clojure it writes.
---

The Architect example starts with one Agent and one task. After reading the
task, the Agent chooses two or three specialist roles, creates them in Clojure,
runs them concurrently, and synthesizes their evidence.

The application does not prescribe the specialist names or roles.

## Run the example

```bash
clojure -M:examples architect \
  "A moon garden's leaves are turning silver after a solar storm. Diagnose the likely causes."
```

The terminal monitor shows the specialists as they are created. A run might
include a botanist, a radiation engineer, and a soil chemist; a different task
can produce a different team.

## Inspect the generated code

`run-architect!` returns a normal Run map. At a REPL, retain it and inspect the
expression the model wrote:

```clojure
(require '[karcarthy :as k])
(load-file "examples/architect/main.clj")

(def result
  (example.architect/run-architect!
   "A moon garden's leaves are turning silver after a solar storm."))

(->> (k/events result)
     (filter #(= :eval/expanded (:type %)))
     (map :expression))
```

Inspect the Agent tree in the same event sequence:

```clojure
(->> (k/events result)
     (filter #(= :agent/started (:type %)))
     (map (juxt :agent :parent-id :depth)))
```

## What this demonstrates

1. The task determines which specialists exist.
2. The model writes control flow, not workflow configuration data.
3. The expression is ordinary Clojure data before evaluation and ordinary
   Clojure code during evaluation.
4. `future` carries the active Run into concurrent Agent calls.
5. Every Agent starts a fresh model conversation while sharing Run-wide limits,
   usage, events, cancellation, and context.

The example deliberately uses one level of generation. Evaluated Agents may
call `eval` recursively when a task genuinely needs another level.

See [Dynamic eval](/docs/eval) for the programming model and [Eval
reference](/docs/reference/eval) for exact events and limits.
