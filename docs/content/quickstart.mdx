This guide creates one Agent, runs it, and inspects the resulting Run.

## Before you begin

You need JDK 21, the Clojure CLI, and an OpenAI API key.

## Add karcarthy

Create `deps.edn`:

```clojure title="deps.edn"
{:deps
 {io.github.soohoonc/karcarthy
  {:git/url "https://github.com/soohoonc/karcarthy"
   :git/sha "985db89bf37bd591939b46cf52cce8c14e72306f"}}
 :paths ["src"]}
```

Set the credential in the shell that will start Clojure:

```bash
export OPENAI_API_KEY="..."
```

## Define the Agent

Create `src/example.clj`:

```clojure title="src/example.clj"
(ns example
  (:require [karcarthy :as k]))

(k/defagent assistant
  {:model "gpt-5.6"
   :instructions "Answer clearly and concisely."
   :input-schema string?
   :output-schema string?})
```

An Agent is a Clojure value containing its model, instructions, Tools, other
Agents it may call, and boundary schemas.

## Run it

Start a REPL:

```bash
clojure -M
```

Require both the example and karcarthy, then start a Run:

```clojure
(require '[example :as example]
         '[karcarthy :as k])

(def run
  (k/run! example/assistant
          "A moon garden's leaves are turning silver. What should we check first?"))
```

The answer varies by model call. Inspect the stable parts of the result:

```clojure
(:status run)          ;=> :completed
(println (:output run))
(:usage run)           ;=> {:model-calls ..., :input-tokens ..., ...}
```

`run!` returns a failed Run for failures during execution. Check `:status`
before consuming `:output`; `:error` explains a failure. Invalid Agent, Tool,
or run configuration throws before execution starts.

## Watch the Run

Create a monitor and pass it as the observer:

```clojure
(def live (k/monitor {:display :tree}))

(def monitored-run
  (k/run! example/assistant
          "List the evidence we should collect before changing the garden."
          {:on-event live}))
```

The terminal redraws while the Agent runs. Evaluate the monitor to print its
final tree again:

```clojure
(k/monitor live)
```

## Change it

Edit the instructions in `src/example.clj`, reload the namespace, and run the
new definition:

```clojure
(require '[example :as example] :reload)
(k/run! example/assistant "Explain it with an example.")
```

You have run the familiar part of karcarthy: one model-backed Agent. The
distinguishing step is next. In [Let an Agent choose the team](/docs/guides/architect),
the garden report causes the Agent to create and run its own specialists with
ordinary Clojure. [Why Clojure?](/docs/why-clojure) explains why that boundary
does not need a workflow DSL.
